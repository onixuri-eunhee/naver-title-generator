## 변경 상세

### 1. 마커 8개 강제

**수정 전:** 마커 지시가 규칙 한 줄에 묻혀 있고, few-shot 예시/체크리스트 없음
**수정 후:**
- ★★★ 강조 별도 섹션으로 분리
- 번호 매긴 few-shot 구조 예시 추가 (← 1번~8번)
- 배치 전략: "도입부 대표이미지 1개 + 나머지 7개를 소제목에 고르게 배치"
- 체크리스트: "□ body 내 마커가 정확히 8개인가? → 8개 미만이면 출력 금지"
- JSON body 필드 설명에 "마커 정확히 8개 포함 필수" 명시

### 2. 글자수 2000자 이상

**수정 전:** 하한선만 제시 ("최소 N자") → AI가 최소값을 목표로 삼음
**수정 후:** "N자 내외 (최소 M자)" 형태로 목표치와 하한 분리

| 프롬프트 | body 수정 전 | body 수정 후 | 전체 목표 |
|----------|-------------|-------------|----------|
| 홈피드(A) | 1300자+ | 1500자 내외 (최소 1400자) | 2200자 내외 (최소 2000자) |
| 네이버SEO(B) | 1000자+ | 1500자 내외 (최소 1400자) | 2200자 내외 (최소 2000자) |
| 구글SEO(C) | 1500자+ | 1700자 내외 (최소 1500자) | 2500자 내외 (최소 2200자) |

- 소제목 4~5개, 각 파트 350자+ 통일

### 3. AI 탐지 회피

**수정 전:** 금지 목록 60개+ (단어 24개 + 접속사 19개 + 패턴 12개)
**수정 후:**

| 항목 | 수정 전 | 수정 후 |
|------|---------|---------|
| 금지 단어 | 24개 나열 | 삭제, "~적인 한자어 최대 1회" 규칙으로 대체 |
| 금지 접속사 | 19개 나열 | 삭제, "같은 접속사 2번 이상 금지" 한 줄 |
| 금지 패턴 | 9~12개 | 핵심 3개만 유지 |
| 어미 순서 지시 | "~거든요→~잖아요→~었어요" | 삭제, "같은 종결어미 연속 2문장 금지" |
| 포지티브 원칙 | 없음 | **5개 추가** (1인칭 경험, 구체적 숫자, 감정 표현, 문장 리듬, 독자 말걸기) |

### 4. API 파라미터
- `api/generate.js`: temperature 0.75 추가 (기본 1.0 → 지시 준수와 창의성 균형)
