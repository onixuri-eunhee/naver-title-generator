# 프리미엄 블로그 이미지 생성 규칙

뚝딱툴 프리미엄 이미지 생성기(`blog-image-pro`)의 전체 파이프라인 규칙.
자동화 에이전트에 전달하여 동일한 품질의 블로그 이미지를 생성할 수 있도록 정리함.

---

## 전체 파이프라인 (3단계)

```
블로그 글 입력
  → [1단계] 마커 배치: 글 분석 → 4~8개 이미지 삽입 위치 제안
  → [2단계] 마커 분석: 마커별 이미지 타입 분류 + 모델 라우팅 + 영문 프롬프트 생성
  → [3단계] 이미지 생성: 프롬프트 → 모델별 API 호출 → 이미지 URL 반환
```

---

## 1단계: 마커 배치 (이미지 삽입 위치 제안)

AI에게 블로그 글 전문을 주고, 이미지를 넣을 최적 위치 4~8곳을 제안받는다.

### 배치 규칙

- **첫 번째 마커**: 반드시 대표이미지 (글 도입부 근처)
- 핵심 문단 뒤에 배치 (문장 중간 X)
- 글 전체에 균등하게 분산 (한곳에 몰리지 않게)
- 각 섹션(【1】,【2】 등)에 최소 1개
- 감성적/묘사적 문단 뒤, 주제 전환 지점에 우선 배치
- **최소 4개, 최대 8개**

### 마커 텍스트 작성 규칙

- 한국어, 5~15자
- 블로그 내용에 맞는 **구체적** 시각 묘사 (예: "커피 원두를 볶는 과정", "아늑한 카페 인테리어")
- 대부분은 사진(photo)에 적합한 장면 묘사
- 데이터/통계/비교가 필요한 경우에만 **트리거 키워드** 포함:
  - 차트/그래프: "차트", "그래프", "비교표", "통계표"
  - 흐름도/타임라인: "흐름도", "타임라인", "로드맵", "프로세스"
  - 포스터/배너: "포스터", "배너"
- 특수 타입(차트/흐름도/포스터)은 글당 최대 1~2개

### 출력 포맷

```json
[
  {"text": "한국어 마커 텍스트", "position": "이 마커가 들어갈 위치 설명 (한국어)"},
  ...
]
```

## 2단계: 마커 분석 + 프롬프트 생성

각 마커를 4가지 이미지 타입 중 하나로 분류하고, 최적 AI 모델을 선택한 뒤, 영문 프롬프트를 생성한다.

### 핵심 원칙: PHOTO-FIRST

- **기본값은 항상 photo** (90% 이상)
- 마커 텍스트에 트리거 키워드가 **명시적으로** 있을 때만 다른 타입 사용
- 블로그가 데이터를 다루더라도 마커에 "차트", "그래프" 등이 없으면 photo
- **첫 번째 마커는 무조건 photo** (대표이미지)

### 4가지 이미지 타입 + 모델 라우팅

| 타입 | 모델 | 용도 | 트리거 키워드 |
|------|------|------|---------------|
| `photo` | FLUX Realism (`fal-ai/flux-realism`) | 사진, 배경, 풍경, 음식, 인물, 제품, 인테리어, 사물 | (기본값, 키워드 불필요) |
| `infographic_data` | GPT Image 1.5 high (`gpt-image-1.5`) | 차트, 그래프, 통계, 수치 비교 | 차트, 그래프, 통계표, 비교표, 수치 비교, 데이터 시각화 |
| `infographic_flow` | Nano Banana 2 (`fal-ai/nano-banana-2`) | 타임라인, 로드맵, 단계, 프로세스 | 흐름도, 타임라인, 로드맵, 프로세스 도식, 단계도 |
| `poster` | Nano Banana 2 (`fal-ai/nano-banana-2`) | 포스터, 배너, 공지문 | 포스터, 배너, 공지문 |

### 타입별 프롬프트 규칙

#### photo (FLUX Realism)

- 시네마틱/에디토리얼 사진 스타일로 묘사
- 포함 요소: 소재, 조명, 카메라 앵글, 무드, 색감, 질감, 배치
- 간판/메뉴 등 텍스트가 있는 요소 → blurred background로 처리
- 카메라: overhead, macro, wide-angle, 45-degree, eye-level
- **반드시 프롬프트 끝에 추가**: `, photorealistic, clean composition, no text, no letters, photography style`
- **한국어 텍스트 포함 절대 금지**
- Korean/East Asian aesthetic 유지

#### infographic_data (GPT Image 1.5 high)

차트/그래프 생성 시 반드시 아래 7가지 규칙(A~G)을 **전부** 따라야 한다:

| 규칙 | 내용 |
|------|------|
| **(A) DATA LABELS** | 모든 데이터 포인트에 숫자 레이블 표시 (바 꼭대기, 파이 조각, 라인 노드) |
| **(B) AXIS UNITS** | Y축에 단위 포함 (예: "비용(만원)"), X축 한국어 라벨, 얇은 격자선 |
| **(C) COMPOSITION** | 차트가 이미지의 70% 이상 차지, 빈 공간 최소화 |
| **(D) SOURCE** | 하단에 데이터 출처/연도 (예: "Source: 한국소비자원 2024") |
| **(E) COLOR** | 핵심 데이터는 굵고 채도 높게, 보조 데이터는 회색 톤. 범례는 우측 또는 하단 |
| **(F) PADDING** | 상단 15% + 하단 10% 여백. 제목/차트가 가장자리에 닿지 않도록 |
| **(G) TITLE** | 2단 구조 — 큰 볼드 한국어 메인 제목 + 작은 부제 (연도/범위) |

- 레이아웃: **2:3 세로형** (1024×1536)
- 프롬프트 안 한국어는 큰따옴표 안에만 (차트 제목, 라벨 등)
- "no text" 접미사 **붙이지 않음** (텍스트가 핵심이므로)

#### infographic_flow (Nano Banana 2)

- 위→아래 또는 왼→오른 흐름으로 구성
- 번호 매긴 한국어 라벨로 3~5개 스텝 노드
- 화살표/커넥터로 연결
- 부드러운 그라데이션 배경
- 메인 스텝은 굵고 진한 색, 서브 스텝은 흐린 색
- 한국어 텍스트는 큰따옴표 안에 포함
- "no text" 접미사 **붙이지 않음**

#### poster (Nano Banana 2)

- 큰 중앙 정렬 한국어 헤드라인 (큰따옴표 안에)
- 아래에 보조 부제
- 볼드 타이포그래피, 고대비 배경
- 폰트 스타일 지정 (sans-serif, bold)
- 최대 2~3색
- "no text" 접미사 **붙이지 않음**

### 공통 프롬프트 규칙

1. **프롬프트는 100% 영어** (한국어는 큰따옴표 안 이미지 내 텍스트만)
2. 프롬프트 길이: **80~150 English words**
3. 구체적으로: 구도, 색상, 레이아웃, 조명, 스타일 명시
4. 마커 텍스트 + 앞뒤 문맥을 정확히 반영 (엉뚱한 주제 X)
5. Korean/East Asian aesthetic 유지

### 출력 포맷

```json
[
  {
    "type": "photo|infographic_data|infographic_flow|poster",
    "model": "fluxr|gpth|nb2",
    "reason": "한국어 1문장 (분류 이유)",
    "prompt": "영어 전용 80-150 words"
  },
  ...
]
```

## 3단계: 이미지 생성 API 호출

### 모델별 API 사양

| 모델 코드 | API | 이미지 크기 | 비고 |
|-----------|-----|------------|------|
| `fluxr` | `fal-ai/flux-realism` | 1024×1024 (정사각) | inference steps: 28, guidance: 3.5, timeout: 30초 |
| `gpth` | OpenAI `gpt-image-1.5` (quality: high) | 1024×1536 (세로 2:3) | output: webp, timeout: 60초 |
| `nb2` | `fal-ai/nano-banana-2` | 1024×1024 (정사각) | timeout: 30초 |

### 후처리 안전장치

- 잘못된 type → `photo`로 강제 변환
- model이 type과 불일치 → 올바른 model로 강제 보정
- prompt 누락 → 기본 photo 프롬프트 적용
- 첫 마커가 photo가 아니면 → photo로 강제 전환

## 분위기 프리셋 (mood)

마커 없이 직접 생성할 때 사용하는 분위기 프리셋:

| 키 | 프롬프트 접두사 |
|----|----------------|
| `bright` | bright, clean, minimal Korean lifestyle blog image, white background, natural daylight, high quality |
| `warm` | warm, cozy, soft tones Korean lifestyle blog image, golden hour lighting, high quality |
| `professional` | professional, corporate, clean Korean business blog image, modern office, high quality |
| `emotional` | emotional, moody, aesthetic Korean blog image, soft bokeh, film tone, high quality |

## 썸네일 텍스트 오버레이 (Canvas 합성)

첫 번째 이미지(대표이미지)에 사용자가 입력한 텍스트를 Canvas로 합성한다.
이미지 자체에는 텍스트가 없고(photo 타입 = `no text`), **프론트엔드에서 후처리**로 얹는 구조.

### 적용 조건

- **대표이미지(첫 번째 이미지)에만** 적용
- 썸네일 텍스트는 **선택 사항** — 비주얼이 중요한 업종(인테리어, 음식 등)은 텍스트 없이 이미지만 쓰는 것을 권장
- 텍스트 최대 30자 (`text.substring(0, 30)`)

### 오버레이 렌더링 사양

| 항목 | 값 |
|------|-----|
| 캔버스 크기 | 1024×1024 |
| 배경 그라데이션 | 하단 50%에서 `transparent` → `rgba(0,0,0,0.7)` (하단으로 갈수록 어두워짐) |
| 그라데이션 범위 | `fillRect(0, 0, 1024, 1024)` — 전체 이미지에 적용 |
| 폰트 | `Noto Sans KR`, weight `900` (Black) |
| 폰트 크기 | 한글 10자 기준 최대 `140px`에서 시작, `maxWidth(920px)`에 맞을 때까지 2px씩 축소, 최소 `48px` |
| 텍스트 색상 | `#FFFFFF` (흰색) |
| 텍스트 정렬 | 수평 `center` (x=512), 수직 `middle` (이미지 중앙 기준) |
| 그림자 | `rgba(0,0,0,0.7)`, blur `12px`, offset `3px/3px` |
| 줄바꿈 | 공백 기준 자동 줄바꿈 (`maxWidth: 920px` 초과 시) |
| 줄 간격 | `fontSize × 1.4` |
| 수직 위치 | 전체 텍스트 블록을 이미지 세로 중앙(y=512)에 배치 |

### 줄바꿈(wrapText) 로직

1. 공백(`' '`) 기준으로 단어 분리
2. 단어를 하나씩 이어붙이며 `measureText()` → `maxWidth(920px)` 초과 시 줄바꿈
3. 한 줄이 여전히 초과하면 **글자 단위**로 자름 (한국어 긴 단어 대응)

### 에이전트용 합성 요약

```
1. 대표이미지(photo)를 1024×1024로 생성 (텍스트 없는 깨끗한 사진)
2. 이미지 전체에 하단→상단 검정 그라데이션 오버레이
3. 이미지 중앙에 Noto Sans KR Black 흰색 텍스트 배치
4. 텍스트 그림자로 가독성 확보
5. 긴 텍스트는 자동 줄바꿈, 폰트 크기 자동 축소
```

## 재생성 규칙

- 전체 재생성: 동일 마커에 대해 **다른 구도와 시각적 접근**으로 프롬프트 생성
- 개별 1장 재생성: 해당 마커의 type을 유지하면서 **완전히 다른 구도/앵글/무드**로 생성
- 재생성 시 프롬프트에 `REGENERATION MODE` 지시 추가

---

## 요약: 에이전트에게 전달할 핵심

1. 블로그 글을 읽고 4~8개 이미지 위치를 먼저 정한다
2. 90% 이상은 photo (FLUX Realism). 차트/흐름도/포스터는 마커에 명시적 키워드가 있을 때만
3. 프롬프트는 100% 영어, 80~150 words, 구체적 시각 묘사
4. photo는 반드시 `no text, no letters, photography style`로 끝낸다
5. 차트는 CHART RULES (A)~(G) 전부 준수
6. 첫 이미지는 무조건 photo (대표이미지)
7. 썸네일 텍스트는 이미지 생성 후 Canvas로 합성 (이미지 자체에 텍스트 X, 오버레이로 얹기)
