# 팀 작업 계획

- 팀명: kkirikkiri-analysis-0330-keyword-diagnosis
- 목표: 황금키워드 기능의 문제점 진단 + 개선안 도출 (20년차 네이버 블로그 키워드 전문가 관점)
- 생성 시각: 2026-03-30

## 유저가 보고한 문제점
1. "웨딩" 입력 → 부대찌개, 두찜 등 엉뚱한 키워드 출력 (1차 테스트)
2. 2차 테스트에서 웨딩 키워드 나왔지만 포화도 전부 0
3. 검색수 적당, 경쟁도 높음, 70-75점대 → 황금키워드라 보기 어려움
4. "이 키워드로 글쓰기" 버튼 클릭 시 블로그 글작성기에 키워드가 안 넘어감 (빈칸)

## 팀 구성
| 이름 | 역할 | 모델 | 담당 업무 |
|------|------|------|----------|
| lead | 팀장 | Opus | 계획/배분/검증/통합 |
| keyword-strategist | 키워드 전략가 | Opus | 네이버 생태계 관점 점수 알고리즘 진단 + 개선안 |
| code-analyst | 코드 분석가 | Opus | api/keywords.js + keyword-finder.html 코드 버그/논리 오류 발견 + 개선안 |

## 태스크 목록
- [ ] 태스크 1: 시드 키워드 생성 프롬프트 분석 → keyword-strategist (엉뚱한 키워드 원인)
- [ ] 태스크 2: 포화도 0 문제 원인 분석 → code-analyst (blogCount 데이터 누락 의심)
- [ ] 태스크 3: 점수 알고리즘 진단 → keyword-strategist (100점 배분 적절성)
- [ ] 태스크 4: 경쟁 도구 점수 방식 조사 → keyword-strategist (키워드마스터 등 비교)
- [ ] 태스크 5: "이 키워드로 글쓰기" 버튼 링크 버그 수정안 → code-analyst
- [ ] 태스크 6: 결과 통합 + 최종 리포트 → lead

## 주요 결정사항

### 결정 1: 점수 표현 방식 변경 (유저 확정)
- 숫자 점수(100점 만점) → **등급 라벨** 방식으로 전환
- 점수는 작게 보조 표시, 라벨을 크게 메인 표시
- 4단계: 블루오션(🔵) / 틈새 공략(🟢) / 경쟁 있음(🟡) / 레드오션(🔴)
- 등급 컷 기준은 알고리즘 개선 후 확정 필요
