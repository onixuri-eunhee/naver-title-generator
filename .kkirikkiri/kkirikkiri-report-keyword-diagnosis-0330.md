## 유저가 보고한 문제

1. "웨딩" 입력 → "부대찌개", "두찜" 같은 엉뚱한 키워드 출력
2. 포화도가 전부 0으로 나옴
3. 검색수 적당 + 경쟁도 높음 → 70-75점, 황금키워드라 보기 어려움
4. "이 키워드로 글쓰기" 버튼 클릭 시 블로그 글작성기에 키워드가 안 넘어감

## 진단 결과 요약

### 문제 1: 엉뚱한 키워드 (부대찌개, 두찜)

| 원인 | 위치 | 심각도 |
|------|------|--------|
| 시드 프롬프트가 분야 제약 없이 "타겟이 검색할 법한" 키워드를 넓게 요청 | api/keywords.js:64-68 | 중 |
| 검색광고 API의 연관키워드가 연쇄 확장 (웨딩→맛집→부대찌개) | api/keywords.js:172-177 | **심각** |
| 자동완성 확장도 분야 필터 없이 인기 검색어 무차별 추가 | api/keywords.js (expandWithAutoComplete) | 중 |
| **분야 적합성 필터가 완전히 부재** — 검색량 50 이상이면 무조건 통과 | api/keywords.js:394 | **심각** |

**수정 방향**: 검색광고 API 결과에 분야 적합도 필터(`isRelevantToSeeds()`) 추가 — 시드 키워드의 핵심 단어(2글자+)를 하나라도 포함하는 키워드만 통과

### 문제 2: 포화도 전부 0

| 원인 | 위치 | 심각도 |
|------|------|--------|
| 상위 40개만 blogCount API 호출, 나머지는 0이 기본값 | api/keywords.js:410-411, 421 | **심각** |
| blogCount=0 → saturation=0 → 포화도 20점 만점 (미수집=블루오션 오판) | api/keywords.js:303-304 | **심각** |
| fetchBlogCount 실패 시 조용히 0 반환 (에러 로그 없음) | api/keywords.js:206, 210 | 중 |
| rate limit 걸리면 상위 40개도 전부 0 → "포화도 전부 0" 직접 원인 | api/keywords.js:206 | **심각** |

**수정 방향**:
- A안(권장): fetchBlogCount 실패 시 -1 반환, 미수집이면 포화도 중립값 부여
- B안(간단): blogCount 조회를 80개 전체로 확대 (14-24초 → 여유 충분)

### 문제 3: 80점이 구조적으로 불가능

| 원인 | 설명 |
|------|------|
| 경쟁도 cliff 구조 | low=20, medium=10, high=3 단 3단계. 네이버 API 경쟁도는 대부분 medium/high |
| 트렌드 대부분 stable | rising=20, stable=10. 대부분 stable이라 -10점 |
| 검색량 100~500 저평가 | 실제 황금 구간인데 22점에 갇힘 |
| 현실적 상한선 ~75점 | 경쟁도 medium + 트렌드 stable이면 최대 75점 |

### 문제 4: "이 키워드로 글쓰기" 버튼 버그

| 원인 | 위치 |
|------|------|
| keyword-finder.html에서 `blog-writer.html?topic=키워드`로 전달 | keyword-finder.html:537, 552 |
| **blog-writer.html에 URLSearchParams 수신 코드가 아예 없음** | blog-writer.html (0건) |

## 개선안

### 1. 점수 표현: 숫자 → 등급 라벨 (유저 확정)

```
🔵 블루오션   — "지금 바로 글 쓰세요!"
🟢 틈새 공략  — "충분히 도전할 만해요"
🟡 경쟁 있음  — "글 품질이 좋아야 해요"
🔴 레드오션   — "상위 노출 어려워요"
```

점수는 작게 보조 표시, 등급 라벨을 크게 메인 표시.

### 2. 점수 배분 재설계 (100점 기준, 등급 산정용)

```
현재:   검색량(30) + 경쟁도(20) + 포화도(20) + 트렌드(20) + 보너스(10)
개선:   검색량(25) + 포화도(30) + 경쟁도(15) + 트렌드(15) + 보너스(15)
```

| 항목 | 현재→개선 | 이유 |
|------|----------|------|
| 검색량 | 30→25 | 황금키워드 핵심은 "틈새". 구간도 6단계로 세분화 |
| 포화도 | 20→**30** | 가장 중요한 지표. 발행량/검색량 비율이 황금키워드의 핵심 |
| 경쟁도 | 20→15 | 검색광고 경쟁도는 블로그 경쟁과 직접 관련 낮음 |
| 트렌드 | 20→15 | stable이 대다수인 현실 반영. 5단계로 세분화 |
| 보너스 | 10→15 | 모바일 비율, 질문형, 롱테일 등 실용적 가산점 확대 |

### 3. 등급 컷 기준 (제안)

| 등급 | 점수 범위 | 설명 |
|------|----------|------|
| 🔵 블루오션 | 70점 이상 | 포화도 낮고, 검색량 있고, 경쟁 적음 |
| 🟢 틈새 공략 | 55~69점 | 도전할 만한 키워드 |
| 🟡 경쟁 있음 | 40~54점 | 글 품질로 승부 필요 |
| 🔴 레드오션 | 39점 이하 | 상위 노출 어려움 |

> 새 알고리즘에서는 현실적으로 블루오션(70+)이 나올 수 있는 구조

### 4. 검색량 구간 개선 (25점)

```
현재                          개선
< 100:       8점              50~200:       15점 (초소형)
100~999:    22점              200~500:      22점 (소형: 초보 황금구간)
1000~4999:  30점              500~2,000:    25점 (중형: 이상적 황금구간)
5000~9999:  25점              2,000~5,000:  20점 (중대형)
10000+:     18점              5,000~10,000: 15점 (대형)
                              10,000+:      10점 (초대형: 대형블로거 독점)
```

### 5. 포화도 구간 개선 (30점)

```
데이터없음(미조회/실패) → 10점 (불확실, 중간값)
saturation ≤ 3          → 30점 (완전 블루오션)
saturation ≤ 8          → 25점 (블루오션)
saturation ≤ 15         → 18점 (적정 경쟁)
saturation ≤ 30         → 10점 (경쟁 있음)
saturation ≤ 50         → 5점  (과포화)
saturation > 50         → 0점  (레드오션)
```

### 6. 경쟁도 개선 (15점)

```
low    → 15점
medium → 8점
high   → 2점
```

### 7. 트렌드 개선 (15점)

```
데이터없음(미조회) → 7점 (중간값)
급상승(+20%+)     → 15점
상승(+10~20%)    → 12점
유지(-10~+10%)   → 7점
하락(-10~-20%)   → 3점
급하락(-20%-)    → 0점
```

### 8. 보너스 개선 (15점, 최대 cap)

```
질문형 키워드 (하는 법, 방법, 추천, 비교)  → +4점
롱테일 (3어절 이상)                       → +3점
모바일 비율 70% 이상                      → +4점
PC/모바일 합산 검색수 균형                 → +2점
복합 해당 (2가지+)                        → +2점
```

### 9. 분야 적합도 필터 (신규)

검색광고 API 연관키워드에서 엉뚱한 키워드를 걸러내는 필터:

```javascript
function isRelevantToSeeds(keyword, seeds) {
  // 시드에서 핵심 단어 추출 (2글자 이상)
  const coreWords = extractCoreWords(seeds);
  // 키워드가 핵심 단어 중 하나라도 포함하면 통과
  return coreWords.some(w => keyword.includes(w));
}
```

### 10. "이 키워드로 글쓰기" 버튼 수정

blog-writer.html에 URL 파라미터 수신 코드 추가:

```javascript
(function() {
  var params = new URLSearchParams(window.location.search);
  var topicFromUrl = params.get('topic');
  if (topicFromUrl) {
    var topicInput = document.getElementById('topic');
    if (topicInput) topicInput.value = topicFromUrl;
  }
})();
```

### 11. API 커버리지 확대

| 데이터 | 현재 | 개선 | 추가 시간 |
|--------|------|------|----------|
| blogCount | 상위 40개 (50%) | 80개 전체 (100%) | +4초 |
| 트렌드 | 상위 25개 (31%) | 40개 (50%) | +2초 |
| 총 실행시간 | 14-24초 | 20-30초 | maxDuration 60초 내 |
