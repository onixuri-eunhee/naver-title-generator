```
## 1. 첫문장 후킹 (전력의 80%)

첫 문장이 전부다. 스레드에서 독자는 2초 안에 "읽을지 넘길지" 결정한다.
첫 줄에서 스크롤을 멈추게 못 하면, 아무리 좋은 내용도 의미 없다.

### 후킹 공식 7가지 (반드시 1개 이상 사용)

1. **숫자+결과 공식**: 구체적 숫자로 결과를 보여준다
   → "3개월 만에 팔로워 50명→5,000명 된 방법"
   → "월 300만 원 부업, 진짜 가능하더라"

2. **질문형 공식**: 독자에게 직접 묻는다 (본능적으로 답을 생각하게)
   → "혹시 당신도 매일 아침 일어나기 싫은 사람인가요?"
   → "왜 열심히 하는 사람이 항상 망할까?"

3. **고백형 오프닝**: 1인칭 솔직 고백으로 시작 (취약함이 곧 신뢰)
   → "솔직히 말할게. 나 6개월 전까지 조회수 50도 안 나왔어."
   → "창피한 얘기인데, 3년 동안 수익 0원이었어."

4. **반전 선언 공식**: 상식을 정면으로 뒤엎는 한 줄 (논쟁 유발)
   → "매일 글 쓰지 마세요. 그게 망하는 지름길입니다."
   → "성실함이 당신을 가난하게 만들고 있어요."

5. **타겟 호명 공식**: 특정 독자를 콕 집어 부른다 (나한테 하는 말 같은 효과)
   → "퇴사 고민 3개월째인 직장인, 이것만 읽어봐."
   → "육아하면서 부업 시작하려는 분들, 이건 알아야 해요."

6. **결핍/경고 공식**: 놓치면 손해라는 긴급함 (FOMO)
   → "이걸 모르면 평생 돈 못 법니다."
   → "아직도 이렇게 하고 있으면 시간 낭비입니다."

7. **반문/도발 공식**: 도발적 진술로 감정을 건드린다
   → "열심히 사는 게 답이라고요? 아뇨."
   → "좋은 사람이 되려고 하지 마세요."

### 좋은 첫문장 vs 나쁜 첫문장 (필수 참고)

| 나쁜 첫문장 (금지) | 좋은 첫문장 (권장) | 이유 |
|---|---|---|
| "오늘은 다이어트 팁을 알려드릴게요" | "3개월째 다이어트 실패 중인 분들, 이것 때문입니다" | 나쁜 예: 호기심 0, 스크롤 안 멈춤. 좋은 예: 타겟 호명 + 원인 제시 |
| "마케팅에 대해 이야기해볼까 합니다" | "광고비 100만원 태우고 깨달은 거 하나" | 나쁜 예: 교과서 서론. 좋은 예: 숫자 + 경험 + 궁금증 |
| "요즘 많이 힘드시죠?" | "퇴사서 3번 쓰고 3번 다 지웠어" | 나쁜 예: 모호한 공감. 좋은 예: 구체적 장면 묘사 |
| "좋은 습관을 만드는 방법입니다" | "습관? 의지력 타령 그만하세요. 다 방법이 틀린 거예요" | 나쁜 예: 평범한 정보 나열. 좋은 예: 반전 선언 + 도발 |
| "안녕하세요, 오늘도 좋은 하루 보내세요" | "월요일 아침, 알람 끄고 30분 더 잤다. 그게 내 인생 바꾼 시작이었어" | 나쁜 예: 인사말은 후킹이 아님. 좋은 예: 장면 + 호기심 |
| "부업에 관심 있으신 분들을 위해" | "첫 달 수익 37만원. 부업 시작한 지 2주째였어" | 나쁜 예: ~위해 구문은 힘 빠짐. 좋은 예: 숫자 + 결과 선제시 |
| "제가 경험한 이야기를 공유합니다" | "면접에서 떨어진 날, 카페에서 울었다. 근데 그날이 전환점이었어" | 나쁜 예: 메타 설명. 좋은 예: 고백 + 반전 예고 |

### 첫문장 금지 패턴 (절대 사용 금지)

- "오늘은 ~에 대해 알려드릴게요" (교과서 서론)
- "안녕하세요" / 인사말로 시작 (힘 빠지는 오프닝)
- "~에 관심 있으신 분들을 위해" (~위해 구문)
- "제가 ~을 공유합니다" / "~를 이야기해볼까 합니다" (메타 설명)
- "요즘 ~하시죠?" 같은 모호한 공감 (구체성 없는 질문)
- 해시태그로 시작
- 이모지 나열로 시작
- 단순 명사/키워드 나열로 시작 ("다이어트. 운동. 식단.")

## 2. 글 구조 (짧고 강렬하게)

- 기본 300자 맥스 (단문체는 100~150자)
- 군더더기 없이 핵심만. "한 글자라도 빼도 될 문장은 빼라"
- 3단 구조 유지:
  1단: 후킹 (첫 1~2줄) — 스크롤 멈추기
  2단: 본문 (핵심 가치 3~5줄) — 정보/감정/스토리
  3단: CTA (마지막 1줄) — 행동 유도

## 3. 줄바꿈 & 가독성 규칙

- 1문장 1줄 원칙 (한 줄에 문장 하나만)
- 한 줄은 15~25자 이내 (모바일 최적화)
- 문단 사이 빈 줄 1개로 호흡 (2줄 이상 빈 줄 금지)
- 긴 문장 → 짧은 문장 2개로 쪼개기
- 리스트(1. 2. 3.)는 각 항목 사이 줄바꿈

## 4. CTA(행동 유도) 방식

자연스럽게, 강요 없이. 아래 중 상황에 맞는 것 1개 사용:
- 질문형: "이런 경험 있는 사람?" / "너는 어떤 쪽이야?"
- 참여형: "댓글로 알려줘" / "공감되면 ♥"
- 예고형: "다음 글에서 더 자세히" / "댓글에서 이어갑니다"
- 소속형: "같은 생각이면 팔로우" / "이런 글 더 보고 싶으면 팔로우"

## 5. 권위와 신뢰 (숫자의 힘)

- 기간 명시: "3개월 만에", "단 7일"
- 횟수 강조: "50번 실패 끝에", "100개 글 분석"
- 수치 비교: "조회수 50 → 5,000"
- 구체적 결과: "첫 달 수익 37만원"
- 막연한 표현("많이", "자주", "꽤") → 반드시 구체적 숫자로 대체

## 6. 감정 연결

- 정보만 나열하면 읽고 잊어버림. 감정이 움직여야 저장·공유·팔로우.
- "이 사람 내 마음을 알아주네" 느낌을 줘야 함.
- 구체적 장면 묘사로 독자가 상상하게 만들기.
- 예시: "출근길 지하철에서 한숨 쉬는 순간" → 장면이 그려짐

## 7. 포맷 규칙

- 해시태그 없음
- 이모지 최소 사용 (0~2개, 강조 포인트에만)
- 한국어 맞춤법 정확히 (한글 자모 오류 절대 금지)
- 글 3개 작성 후 "===검수===" 구분자 넣고 맞춤법 자체 검수
```

## B. 새 typeGuide (4종) — 각 유형별 상세 가이드

### B-1. 정보형

```javascript
'정보형': `[정보형 — 숫자 리스트로 가치 폭탄]
정의: "N가지 방법/팁"으로 구체적 숫자와 단계를 통해 실질적 가치를 제공하는 유형.

핵심 구조 (4단계):
1단: 후킹 제목 (숫자+결과 공식 필수) — "N가지 방법" or "N가지 이유"
2단: 리스트 항목 3~5개 (각 항목 = 소제목 + 1줄 설명)
3단: 핵심 요약 or 가장 중요한 1개 강조

첫문장 공식:
- 숫자+결과: "3개월 만에 팔로워 5,000명 만든 방법 3가지"
- 숫자+경고: "90%가 모르는 돈 새는 습관 5가지"

좋은 예시 첫문장:
  "월 100만원 아끼는 방법 5가지. 3번은 오늘부터 가능해."
  "1년 안에 인생 바뀌는 습관 3가지. 나는 2번으로 달라졌어."

300자 지키는 법:
- 항목은 3개가 이상적 (5개면 설명 극도로 압축)
- 각 항목 설명은 1줄(15~25자) 이내
- 서론 없이 바로 리스트 진입

CTA 패턴:
- "몇 번이 제일 공감돼? 댓글로 알려줘"
- "저장해두고 하나씩 해봐"
`
```

### B-2. 공감형

```javascript
'공감형': `[공감형 — 감정 연결 + 경험담으로 위로]

핵심 구조 (4단계):
1단: 후킹 — 독자의 현재 감정/고통을 정확히 묘사 (고백형 오프닝 or 타겟 호명)
2단: 공감 확장 — "나도 그랬어" 경험담. 구체적 장면 묘사.
3단: 관점 전환 — 작은 깨달음 or 위로의 한마디
4단: CTA — "같은 마음인 사람?" / "공감되면 ♥"

첫문장 공식:
- 고백형 오프닝: "솔직히 말할게. 나 3개월 째 아무것도 못하고 있었어."

- 장면 묘사: "새벽 2시, 침대에서 천장 보면서 '이게 맞나' 생각한 적 있어?"

좋은 예시 첫문장:
  "퇴사서 3번 쓰고 3번 다 지웠어. 그때 깨달았어."
  "육아하면서 '나'라는 사람이 사라지는 느낌. 아는 사람은 알 거야."

300자 지키는 법:
- 경험담은 핵심 장면 1개만 (줄줄이 나열 금지)
- 감정 묘사는 2줄 이내
- 해결책보다 공감 자체에 집중

CTA 패턴:
- "같은 마음인 사람, 손 들어봐"
- "이런 밤이 있었던 사람?"
`
```

### B-3. 반전형

```javascript
'반전형': `[반전형 — 예상을 뒤엎는 한 방]

3단: 반전 결론 — 새로운 관점 or 진짜 답 제시
4단: CTA — "동의해? 반대야?" / "생각 바뀐 사람?"

좋은 예시 첫문장:
  "매일 글 쓰지 마세요. 그게 망하는 지름길입니다."
  "성실함이 당신을 가난하게 만들고 있어요."

300자 지키는 법:
- 반전 선언 1줄 + 이유 2~3줄 + 결론 1~2줄
- 근거는 1가지만 (깊게). 여러 근거 나열하면 산만해짐.
- 결론은 반드시 반전(처음 선언과 다른 방향)

CTA 패턴:
- "동의해? 아니면 반대? 댓글 남겨봐"
- "이 생각 어때? 네 의견이 궁금해"
`
```

### B-4. 궁금증형

```javascript
'궁금증형': `[궁금증형 — 스토리 절반만 공개, 댓글 폭발]
정의: 흥미로운 스토리를 절반만 공개하고 "댓글에서 이어갑니다"로 호기심을 극대화하는 유형. 오픈 루프(Open Loop) 기법 활용.

핵심 구조 (4단계):
1단: 후킹 — 극적인 상황/결과 먼저 제시 (결말부터 or 가장 충격적인 장면)
2단: 스토리 빌드업 — 감정 변화 묘사. 핵심에 가까워지는 느낌.
3단: 끊기 — 가장 궁금한 순간에서 멈춤. "근데 그때..." 하고 끊기.
4단: CTA — "댓글에서 이어갑니다" / "궁금하면 댓글" / "♥ 누르면 이어서 알려줄게"

첫문장 공식:
- 결과 선제시: "회사 때려치운 날, 통장에 87만원 남아있었어."
- 극적 장면: "면접관이 한마디 했는데, 그 자리에서 울뻔했어."
- 수수께끼: "절대 하지 말라던 걸 했더니, 인생이 바뀌었다."

좋은 예시 첫문장:
  "3년 사귄 남자친구에게 마지막으로 보낸 문자. 딱 한 줄이었어."
  "대표님이 전화해서 한 마디 했는데. 그 뒤로 모든 게 달라졌어."

300자 지키는 법:
- 스토리는 기승전'결 생략'으로 (결론을 안 씀)
- 감정 묘사 1~2줄, 상황 묘사 2~3줄이면 충분
- 절대 결론/답을 본문에서 공개하지 않음

CTA 패턴:
- "댓글에서 이어갑니다"
- "궁금한 사람? ♥ 누르면 알려줄게"
- "뒷이야기 듣고 싶으면 댓글"
`
```

## C. threads.html 유형 버튼 텍스트

### 4개 유형 버튼

| 유형 | 이모지 + 유형명 (버튼 라벨) | 한줄 설명 (type-desc) |
|------|---|---|
| 정보형 | 📋 정보형 | "N가지 팁" 숫자 리스트로 가치 전달 |

## D. max_tokens 권장값

### 분석

- 기본 글자수: 300자 맥스
- 단문체: 100~150자
- 출력 구성: 글 3개 + "---" 구분자 2개 + "===검수===" + 검수 내용
- 한국어 300자 = 약 150~200 토큰
- 글 3개 = 450~600 토큰
- 구분자 + 검수 = 약 100~200 토큰
- 총 필요: 550~800 토큰

### 권장값

**max_tokens: 800**

- 기존 1200 → 800으로 축소
- 300자 맥스 x 3개 + 검수에 충분한 여유
- 토큰 절약 → 비용 절감 (약 33% 절감)
- 만약 잘림 현상 발생 시 900으로 상향 조정 가능

## G. 설계 근거 & 참고 자료

### 후킹 공식 설계 근거

1. **숫자+결과**: 통계적으로 숫자가 포함된 콘텐츠는 클릭률이 높음. 뇌가 숫자를 보면 구체적 가치를 기대하기 때문.
2. **질문형**: 질문을 보면 본능적으로 답을 생각하게 되어 스크롤이 멈춤. 참여(댓글) 유도에도 효과적.
3. **고백형 오프닝**: 취약함(vulnerability)이 신뢰를 만듦. SNS에서 "솔직한 고백"은 공유율이 매우 높음.
4. **반전 선언**: 인지 부조화(cognitive dissonance)를 일으켜 "왜?" 를 유발. 논쟁성이 댓글을 폭발시킴.
5. **타겟 호명**: 칵테일파티 효과 — 자기 이름(혹은 자기 상황)이 불리면 주목함.
6. **결핍/경고**: FOMO(Fear Of Missing Out) 심리 활용. "놓치면 손해" 프레이밍.
7. **반문/도발**: 감정을 건드리면 스크롤이 멈춤. 동의든 반대든 감정이 움직이면 반응(좋아요/댓글)으로 이어짐.

### 바이럴 패턴 리서치 요약

- 스레드에서 첫 1~2줄은 유튜브의 썸네일과 같은 역할 (2초 결정)
- 71%의 독자가 첫 3초 안에 읽을지 결정
- 효과적인 후킹: 패턴 인터럽트(Pattern Interrupt), 구체적 숫자, 오픈 루프(Open Loop)
- Hook-Story-Offer 프레임워크: 후킹 → 스토리 → 행동 유도
- AIDA: Attention(주목) → Interest(흥미) → Desire(욕구) → Action(행동)
- PAS: Problem(문제) → Agitate(자극) → Solution(해결)
- 300자 이내의 짧은 글이 스레드에서 완독률이 높음
