# 숏폼 벤치마킹 파이프라인 + 통합 동선 재설계

**작성일:** 2026-04-14
**상태:** 설계 승인 대기 (사용자 리뷰 단계)
**연관 컨텍스트:** 자영업자 타겟 / 4/25 정식 오픈 / 카드뉴스 Phase 3 (내 이미지 보관함) 완료 후속

---

## 1. 배경 및 핵심 통찰

### 자영업자 숏폼은 바이럴 숏폼이 아니다

자영업자(웨딩플래너, 매장 사장, 강사, 컨설턴트 등)가 만드는 숏폼은 "100만뷰 도전"이 아니다. **100명이 보고 그 중 5명이 매장에 오면 성공**이다. 이게 시스템 설계 전체를 좌우한다.

- ❌ "조회수 100만 영상" 패턴 따라하기 → 사장님이 못 따라하는 자극적 후킹, 화려한 편집
- ❌ AI 양산 채널 패턴 → 신뢰 깎임, 유튜브 차단 위험
- ❌ 일반 인플루언서 패턴 → "전문가가 아니라 그냥 인플루언서"
- ✅ **동종 업계 자영업자 중 작지만 잘 된 영상 패턴**

### 기획 70% 철학

뚝딱툴은 자동화 도구지만, **기획에 70% 이상 투입**해야 살아남는다. 자영업자가 자기 업을 알리고 고객을 유치해야 우리도 산다. 무지성 자동 생성보다 검증된 패턴 기반 생성이 장기 가치다.

### 현 상태 (2026-04-14)

- 숏폼 도구 존재, ElevenLabs 통합 완료, 키네틱 타이포 프리셋 10종 리브랜딩 완료
- Phase 2 (효과 풀) 미구현
- 카드뉴스 Phase 3 (내 이미지 보관함) 완료 — 같은 보관함을 숏폼이 재사용 가능

### 목표

자영업자가 **본인의 블로그 글 또는 키워드 + 경험·느낌 + 정체성**을 입력하면, 시스템이 자동으로:
1. YouTube에서 동종 키워드의 "작지만 잘된" 영상을 찾고
2. Gemini Vertex AI로 패턴 분석하고
3. 그 패턴을 적용해 Claude Opus가 대본 작성하고
4. 키네틱 타이포 메인 + 사용자 사진 액센트로 영상 생성하고
5. 같은 패턴의 YouTube SEO 캡션까지 만들어준다

## 2. 핵심 통찰 정리 (스펙 전체에 적용되는 원칙)

### 원칙 1: 비주얼 메인 = Remotion 키네틱 타이포
사진/AI 이미지는 **20% 이내 액센트**일 뿐. 메인은 키네틱 타이포 + 다이내믹 효과(Phase 2). 이유:
- 즉시 렌더링 (AI 대기 없음)
- 브랜드 일관성 보장
- AI 이미지 과다 = "AI 양산" 느낌 → 신뢰 손실

### 원칙 2: 벤치마킹은 끝까지 일관성
같은 벤치마킹 결과가 **대본 + 캡션 + 비주얼 + BGM 분위기 추천**까지 모두 적용된다. 패턴이 패키지 전체를 관통.

### 원칙 3: 사용자 통제권 + 투명성
- AI가 자동으로 결정한 것은 모두 "왜 이렇게 됐는지" 설명한다 ("벤치마킹 영상 3개에서 추출")
- 사용자는 모든 단계에서 수정/재선택 가능
- 단, 디폴트는 항상 "다음 누르면 잘 될 것"

### 원칙 4: 페르소나 일관성
사용자가 Step 1에서 본인 정체성(블로거/사장/강사 등)과 톤(전문가/친근)을 선택. 모든 산출물(대본/캡션)이 그 페르소나의 1인칭으로 작성됨. AI 양산 느낌 차단.

### 원칙 5: 자막 이모지 절대 금지
자영업자 전문성/신뢰감 손상. Claude 프롬프트에 hard rule + 후처리 검증으로 이중 차단.

## 4. 사용자 플로우

### Step 1 — 입력 (한 화면)

```
┌─ 콘텐츠 ──────────────────────────────┐
│ ○ 블로그 글 사용                       │
│   [텍스트 영역 — /blog-writer 자동 입력] │
│ ○ 키워드만 사용                        │
│   [키워드 입력]                         │
└────────────────────────────────────────┘

┌─ 내 경험·느낌 (필수) ─────────────────┐
│ [자유 텍스트 — 본인의 구체적 경험]      │
└────────────────────────────────────────┘

┌─ 내 정체성 ──────────────────────────┐
│ 화자: [매장 사장 / 블로거 / 강사 /      │
│        컨설턴트 / 프리랜서 /            │
│        직접 입력]                       │
│ 톤: ( ● 전문가  ○ 친근한 친구 )         │
└────────────────────────────────────────┘

화자 페르소나 5종 (+직접 입력):
1. 매장 사장
2. 블로거
3. 강사
4. 컨설턴트
5. 프리랜서
6. 직접 입력 (자유)

(디자이너는 MVP에서 제외 — 프리랜서로 흡수 가능.
필요 시 후속 스프린트에서 확장)

┌─ 영상 길이 ─────────────────────────┐
│ ○ 30초  ● 45초  ○ 60초  ○ 90초      │
└──────────────────────────────────────┘

[ 다음: 벤치마킹 영상 찾기 → ]
```

**검증:**
- 블로그 글 또는 키워드 둘 중 하나는 필수
- 내 경험·느낌 필수 (50자 이상)
- 화자 페르소나 + 톤 선택 필수

### Step 2 — 벤치마킹 (시스템 자동, 약 30~45초)

서버에서 다음 작업이 진행되는 동안 진행률 표시:

1. **검색 키워드 추출** (Gemini Flash, ~2초)
   - 블로그 글에서 핵심 명사·의도 추출
   - 또는 사용자 키워드 그대로 사용
   - 결과: 3~5개 검색어 조합

2. **YouTube 검색 + 필터** (~5초)
   - YouTube Data API search.list + videos.list + channels.list
   - 필터:
     - 한국어 (regionCode=KR, relevanceLanguage=ko)
     - 영상 길이 ≤ 90초
     - 자막 보유 (videoCaption=closedCaption)
     - 업로드 시점 ≤ 12개월
     - 조회수 ≥ 10,000
     - 조회수 / 구독자 비율 ≥ 5
   - 결과: **정확히 5개** 후보 영상 (필터 통과한 것 중 viewToSubRatio 내림차순 상위 5)

3. **결과 카드 표시** (사용자 화면)
   - 각 카드:
     - 썸네일
     - 채널명
     - 조회수 / 구독자수 / 길이
     - "왜 추천?" 1줄 (예: "구독 8천에 12만 조회 — 8x 비율")
     - 첫 5초 자막 미리보기 (있으면)
   - 사용자가 1~3개 체크박스로 선택
   - "벤치마킹 없이 진행"도 가능 (폴백 모드)

4. **선택된 영상 깊은 분석** (Gemini Vertex AI, ~20초)
   - 선택된 1~3개 영상 URL을 Gemini에 직접 전달
   - 멀티모달 분석 (영상 + 음성 + 자막)
   - 출력: 구조화된 JSON (Schema 5절 참고)

5. **패턴 카드 1장 표시**
   - aggregated 패턴 한국어 요약
   - "이 패턴으로 대본 작성할게요" 확인 → Step 3

### Step 3 — 대본 생성 + 인라인 편집

- Claude Opus가 다음 입력으로 대본 생성:
  - 사용자 블로그 글 (베이스 콘텐츠)
  - 사용자 경험·느낌 (개인화 양념)
  - Gemini JSON (구조 가이드)
  - 화자 페르소나 + 톤
- 출력: 대본 JSON (scenes 배열)
- 사용자에게 표시: 텍스트 형태로 인라인 편집 가능
- 편집 후 "다음" 클릭 → Step 4

### Step 4 — 음성 + BGM

기존 숏폼 도구의 음성 단계 재사용. 단, BGM 선택 옵션 추가:
- 본인 음성 업로드 또는 TTS 선택 (Supertone/ElevenLabs)
- 미리듣기
- BGM: 카테고리 선택 (활기찬/차분/감성/일상/없음)

### Step 5 — 비주얼 액센트 (간단)

```
┌─────────────────────────────────────────┐
│ 메인 영상은 키네틱 타이포로 자동 생성돼요│
│ 핵심 순간에 사진/이미지를 양념으로 추가  │
│ 하면 더 풍성해져요. (선택사항)            │
└─────────────────────────────────────────┘

┌─ 내 사진 (보관함에서) ─┐
│ [+ 사진 선택] 0/3장    │
│ 추천: 매장 사진, 본인  │
│ 사진, 메뉴/상품 사진    │
└────────────────────────┘

┌─ AI 이미지 (자동 생성) ─┐
│ ○ 사용 안 함            │
│ ● 1장 생성              │
│ ○ 2장 생성              │
└─────────────────────────┘

[ 다음: 미리보기 → ]
```

- 0~3장 사진 + 0~2장 AI 이미지
- 사진은 카드뉴스에서 만든 `<ImagePickerModal>` 재사용 (보관함 통합)
- AI 이미지는 기존 FLUX Schnell 파이프라인 재사용

### Step 6 — 미리보기 + 커스터마이징 ⭐ (가장 비중 큰 단계)

**진입 시 자동 추천:**
```
┌────────────────────────────────────────┐
│ 벤치마킹 영상 패턴 분석:                 │
│ → 추천 프리셋: 친근 모드 ⭐              │
│                                          │
│ [ 추천대로 갈게 ]  [ 세부 조정 ]         │
└────────────────────────────────────────┘
```

**프리셋 6종 (이번 스펙에 정의):**
| 프리셋 | 키네틱 스타일 | 텍스트 위치 | 카메라 모션 | 씬 전환 | 자막 색 | BGM 카테고리 |
|---|---|---|---|---|---|---|
| 전문가 | 정적 | 하단 | 정적 | 페이드 | 흰색 | 차분 |
| 친근 | 키네틱(라이트) | 중앙 | Ken Burns | 슬라이드 | 옐로우 | 활기찬 |
| 임팩트 | 키네틱(헤비) | 중앙(큰) | 줌 인 | 컷 | 빨강 | 임팩트 |
| 차분 | 정적 | 하단 | 정적 | 페이드(긴) | 베이지 | 감성 |
| 트렌디 | 키네틱(워드별) | 자유 | 패닝 | 슬라이드(빠름) | 형광 | 트렌드 |
| 비즈니스 | 정적 | 상단 | 정적 | 컷 | 네이비 | 차분 |

**세부 조정 모드 (펼침):**
- 텍스트 위치: 상단/중앙/하단
- 카메라 모션: 정적/Ken Burns/줌/패닝
- 씬 전환: 컷/페이드/슬라이드
- 자막 폰트: 5종 (Pretendard/Noto Sans/Spoqa/IBM Plex/Suit)
- 자막 색: 8종 + 직접 입력 (HEX)
- 자막 크기: 슬라이더 (24~96px)
- 자막 배경색: 8종 + 투명도 슬라이더
- BGM 카테고리: 5종 (활기찬/차분/감성/일상/임팩트) + 무음
- 액센트 위치 조정: 사진을 어느 씬에 배치할지 드래그

**Remotion Player 실시간 미리보기:**
- 모든 변경이 즉시 반영
- 재생/일시정지/구간 반복

## 6. Claude Opus 프롬프트 구조

### 시스템 프롬프트

```
당신은 자영업자를 위한 한국어 숏폼 대본 작가입니다.
주어진 블로그 글을 베이스로, 벤치마킹 패턴을 적용해
사용자가 자기 영상으로 만들 수 있는 30~90초 대본을 작성합니다.

## 절대 규칙 (위반 시 결과 폐기)
1. 자막에 이모지 사용 금지 (전문성 손상)
2. 화자: 사용자가 선택한 페르소나의 1인칭 시점 유지
```

## 8. API 설계

### 8.1 신규 엔드포인트

#### POST /api/shortform-benchmark
**역할:** 키워드/블로그 글 → 벤치마킹 후보 영상 **정확히 5개** 반환

**요청:**
```json
{
  "blogText": "...",
  "keywords": ["웨딩플래너", "신랑정장"],
  "limit": 5
}
```

**응답:**
```json
{
  "candidates": [
    {
      "videoId": "abc123",
      "title": "...",
      "thumbnail": "https://...",
      "channelName": "...",
      "viewCount": 120000,
      "subscriberCount": 8000,
      "duration": 35,
      "viewToSubRatio": 15.0,
      "subtitlePreview": "안녕하세요. 오늘은..."
    }
  ],
  "searchKeywords": ["웨딩플래너 신랑 정장", "..."],
  "cached": false
}
```

**로직:**
1. Redis 캐시 확인 (`bench:cache:${sha256(blogText + keywords)}`)
2. 캐시 적중: 즉시 반환
3. 캐시 미스:
   a. Gemini Flash로 검색 키워드 추출
   b. YouTube search.list → videos.list → channels.list
   c. 5배 비율 + 1만 조회 등 필터 적용
   d. 결과 5개 미만 시 필터 완화 (구독자 비율 5x → 3x)
   e. Redis 저장 (TTL 7일)
   f. 응답
4. 인증 필요 (Bearer 토큰)
5. 비용: 사용자 크레딧 차감 없음 (벤치마킹은 무료, 영상 생성 단계에서만 차감)

#### POST /api/shortform-benchmark/analyze
**역할:** 사용자가 선택한 1~3개 영상을 Gemini로 깊게 분석

**요청:**
```json
{
  "videoIds": ["abc123", "def456"],
  "videoUrls": ["https://www.youtube.com/watch?v=abc123", "..."]
}
```

**응답:** Section 5 JSON Schema 그대로

**로직:**
1. 인증 확인
2. Vertex AI Gemini 호출 (fileData = YouTube URL, JSON 출력 강제)
3. 결과 검증 (필수 필드 존재 확인)
4. 응답
5. 비용: Vertex 크레딧 사용 (사용자 크레딧 차감 없음)

#### POST /api/shortform-script (기존 확장)
**역할:** 대본 + 캡션 생성. 기존 엔드포인트에 벤치마킹 분석 결과를 받는 필드 추가.

**요청 (확장):**
```json
{
  "blogText": "...",
  "userExperience": "...",
  "persona": "사장",
  "tone": "친근",
  "duration": 45,
  "benchmarkAggregated": { /* Gemini JSON의 aggregated 필드 */ }
}
```

**응답 (확장):**
```json
{
  "scenes": [...],
  "totalDuration": 45,
  "presetUsed": "친근",
  "caption": "...",
  "captionHashtags": ["#...", "..."]
}
```

## 9. 캐싱 전략

### Redis 키 구조

```
bench:cache:{sha256(blogText + keywords)}    TTL 7일
bench:analyze:{videoId}                       TTL 30일 (영상 자체 분석은 거의 안 변함)
```

### 캐시 적중률 가정

- 인기 키워드 ("웨딩플래너", "헬스장", "카페")는 7일 내 반복 검색 → 적중률 60%+
- 고유 키워드는 캐시 미스 → 매번 신선 검색
- 영상 분석 캐시 (videoId 단위)는 적중률 80%+ (같은 영상이 여러 사용자 결과에 등장)

### 캐시 무효화

- 수동 무효화 엔드포인트 없음 (TTL로만)
- YouTube 정책 변경/특별한 사유 시 Redis FLUSH (운영자 작업)

## 10. YouTube Data API 쿼터 관리

### 1회 검색 비용
- search.list: 100 units
- videos.list (50 영상): 50 units
- channels.list (~30 채널 dedupe): 30 units
- **합계: 180 units / 검색 (캐시 미스 시)**

### 일일 예산

| 사용자 규모 | 일일 검색 추정 | 캐시 적중률 60% | 실제 API 호출 | 쿼터 사용 |
|---|---|---|---|---|
| MVP (50 DAU) | 50회 | 30% 미스 | 15회 | 2,700 units |
| 성장 (200 DAU) | 200회 | 40% 미스 | 80회 | 14,400 units |
| 안정 (500 DAU) | 500회 | 50% 미스 | 250회 | 45,000 units |

**무료 일일 쿼터: 10,000 units → 50 DAU까지는 무료 충분**

### 쿼터 상향 신청

200 DAU 도달 시 Google Cloud Console에서 쿼터 상향 신청. 예상 승인 쿼터: 1,000,000 units/day (사실상 무한). 신청서에 명시할 사유:
- 자영업자 콘텐츠 제작 도구
- 원본 영상 직접 노출 X (메타데이터 + 구조 분석만)
- 1회 사용자 액션당 1 검색 제한

## 11. Gemini Vertex AI 비용

### 1회 영상 분석 비용 (Gemini 2.5 Flash 기준)

- 60초 영상 ≈ 15,500 input tokens (영상 토큰화율 258 tokens/sec)
- 출력 ≈ 500 tokens
- 1건 비용: ~$0.0012 (input) + ~$0.00015 (output) = **약 $0.0014**
- 3건 분석: **약 $0.004**

### 운영자 보유 Vertex AI 크레딧으로 사실상 0원

월 1,000명이 각 1회씩 사용해도 약 $4. 1년 50,000 사용자 = $200. 운영자 보유 크레딧으로 충분.

## 12. 실패 처리

### 시나리오

| 실패 | 사용자 영향 | 폴백 |
|---|---|---|
| YouTube API 쿼터 초과 | 벤치마킹 단계 막힘 | "벤치마킹 없이 진행" 버튼 제공 → Step 3로 점프 |
| 검색 결과 0건 | 후보 카드 없음 | 필터 자동 완화 → 그래도 없으면 폴백 모드 |
| Gemini Vertex AI 다운 | 패턴 분석 실패 | "벤치마킹 없이 진행" 폴백 |
| Claude Opus 응답 실패 | 대본 생성 실패 | 자동 재시도 1회 → 실패 시 사용자에 에러 표시 |
| 사용자가 벤치마킹 영상 0개 선택 | "벤치마킹 없이 진행" 명시적 선택 | 기본 프리셋(친근)로 대본 생성 |
| 자막 없는 영상만 검색됨 | 패턴 분석 품질 저하 | 자동 필터 (자막 있는 영상만) |
| 벤치마킹 필드 누락된 응답 | JSON 파싱 실패 | 누락 필드는 기본값으로 채움 |

### 폴백 모드 (벤치마킹 없이 진행)

벤치마킹을 건너뛴 경우:
- Claude Opus는 사용자 입력만으로 대본 생성
- 프리셋: 사용자 톤 선택에 따라 자동 ("전문가" 톤 → "전문가" 프리셋, "친근" 톤 → "친근" 프리셋)
- 캡션: 일반 SEO 베스트 프랙티스 적용 (해시태그 5개, 200자 내외, 일반론)

| 저작권 / 표절 우려 | 벤치마킹 영상의 대본 직접 인용 | Claude 프롬프트에 "패턴만 참고, 직접 인용 금지" hard rule. 출력 후 유사도 검증 (5문장 연속 일치 시 재생성) |
| 벤치마킹 결과를 사용자가 신뢰 못함 | 단계 이탈 | 각 후보 카드에 "왜 이게 추천됐는지" 1줄 설명 |
| 페르소나 6종이 사용자에게 안 맞음 | 적합한 페르소나 없음 | "직접 입력" 옵션 제공 |

## 19. 성공 기준

- [ ] 자영업자가 키워드 또는 블로그 글 + 본인 정체성 입력 → 60초 내에 대본+영상+캡션 패키지 받음
- [ ] 벤치마킹 영상 5개 이상이 5배 비율 필터를 통과해 표시됨 (95% 케이스)
- [ ] Gemini 패턴 추출 JSON이 schema 검증을 통과 (99% 케이스)
- [ ] 생성된 대본에 이모지 0개, 일반론 표현 ≤ 2회
- [ ] 캡션이 벤치마킹 패턴의 해시태그 수 ±2 이내로 일치
- [ ] Step 6 미리보기에서 모든 커스터마이징이 실시간 반영
- [ ] 폴백 모드 (벤치마킹 없이) 단독으로도 대본 생성 가능
- [ ] 운영자가 직접 자기 사업(웨딩플래너)으로 도그푸드 → "이거 진짜 내가 쓴 것 같아" 만족도 도달
