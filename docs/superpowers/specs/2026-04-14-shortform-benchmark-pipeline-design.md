## 1. 배경 및 핵심 통찰

### 자영업자도 100만뷰 가능. 비결은 "검증된 구조"

자영업자(웨딩플래너, 매장 사장, 강사, 컨설턴트 등)도 릴스/숏츠에서 **100만뷰, 200만뷰가 흔하다**. 비결은:

- **공감 요소로 후킹** → 누구나 끄덕이는 첫 3초
- **본인이 보여주고 싶은 것 자연스럽게 노출** → 매장/메뉴/전문성
- **검증된 터진 구조를 따라감** → 처음부터 새로 만들지 않음

자영업자가 못 만드는 게 아니라 **구조를 모를 뿐**이다. 우리 도구의 미션은 그 구조를 자동으로 분석해서 본인 콘텐츠에 적용하는 것.

### 풀링 콘텐츠 vs 키 콘텐츠 — 두 가지 전략

자영업자의 SNS 운영은 두 종류 콘텐츠가 같이 작동한다:

| 콘텐츠 타입 | 목적 | 특징 | 우리 도구 역할 |
|---|---|---|---|
| **풀링 콘텐츠** ⭐ | 도달/조회수 폭증 (100만뷰 가능) | 검증된 터진 구조 + 공감 후킹 + 키워드 광범위 | **자동 생성 (메인)** |
| **키 콘텐츠** | 신뢰/전환 | 진정성 있는 본인 이야기, 프로필 상단 고정 | 보조 (사용자가 직접 만들고 고정) |

**우리 도구는 풀링 콘텐츠를 잘 만들도록 돕는다.** 사용자가 풀링 콘텐츠로 100만뷰를 받으면 → 그 시청자 일부가 프로필로 와서 키 콘텐츠를 봄 → 신뢰 형성 → 매장 방문/예약. 두 콘텐츠가 같이 작동해야 매출로 이어짐.

### 키워드 확장 + 5x 비율 = "검증된 구조" 자동 추출

초기 설계에서 "동종 업계 자영업자" 한정으로 검색하려 했으나 **모수 부족 문제** (뷰트랩 등 기존 도구 사용 경험 검증). 자영업자 영상은 절대량이 적어 패턴 추출 불가.

**해결:** 우리가 추출하는 건 영상의 "터진 구조"(후킹/본문/CTA/시각/음성 패턴)이지 영상 자체의 콘텐츠가 아님. 따라서 **누가 만들었는지(자영업자 vs 인플루언서)는 덜 중요**. 키워드 + 관련 키워드로 모수를 넓히고 5x 비율 필터로 "터진 구조" 품질 보장.

**5x 비율 필터는 자연스럽게 "채널 평균을 뛰어넘은 폭발"을 잡는다:**

| 케이스 | 구독자 | 조회수 | 비율 | 결과 |
|---|---|---|---|---|
| 작은 채널 breakout | 1천 | 1만 | 10x | ✅ 후보 |
| 중간 채널 viral | 1만 | 10만 | 10x | ✅ 후보 |
| 큰 채널 hyper-viral | 10만 | 100만 | 10x | ✅ 후보 |
| 큰 채널 평균 | 100만 | 200만 | 2x | ❌ 탈락 (평소 정도) |

채널 크기와 무관하게 "그 채널의 평균을 뛰어넘은 영상"만 통과. 즉 **터진 구조가 있는 영상**만 잡힘. 우리가 학습할 정확한 그것.

### 기획 70% 철학

뚝딱툴은 자동화 도구지만, **기획에 70% 이상 투입**해야 살아남는다. 자영업자가 자기 업을 알리고 고객을 유치해야 우리도 산다. 무지성 자동 생성보다 검증된 패턴 기반 생성이 장기 가치다.

### 현 상태 (2026-04-14)

- 숏폼 도구 존재, ElevenLabs 통합 완료, 키네틱 타이포 프리셋 10종 리브랜딩 완료
- Phase 2 (효과 풀) 미구현
- 카드뉴스 Phase 3 (내 이미지 보관함) 완료 — 같은 보관함을 숏폼이 재사용 가능

자영업자가 **본인의 블로그 글 또는 키워드 + 경험·느낌 + 정체성**을 입력하면, 시스템이 자동으로:

## 2. 핵심 통찰 정리 (스펙 전체에 적용되는 원칙)

### 원칙 1: 비주얼 메인 = Remotion 키네틱 타이포
사진/AI 이미지는 **20% 이내 액센트**일 뿐. 메인은 키네틱 타이포 + 다이내믹 효과(Phase 2). 이유:
- 즉시 렌더링 (AI 대기 없음)
- 브랜드 일관성 보장
- AI 이미지 과다 = "AI 양산" 느낌 → 신뢰 손실

### 원칙 2: 벤치마킹은 끝까지 일관성
같은 벤치마킹 결과가 **대본 + 캡션 + 비주얼 + BGM 분위기 추천**까지 모두 적용된다. 패턴이 패키지 전체를 관통.

### 원칙 3: 사용자 통제권 + 투명성
- AI가 자동으로 결정한 것은 모두 "왜 이렇게 됐는지" 설명한다 ("벤치마킹 영상 3개에서 추출")
- 사용자는 모든 단계에서 수정/재선택 가능
- 단, 디폴트는 항상 "다음 누르면 잘 될 것"

### 원칙 4: 페르소나 일관성
사용자가 Step 1에서 본인 정체성(블로거/사장/강사 등)과 톤(전문가/친근)을 선택. 모든 산출물(대본/캡션)이 그 페르소나의 1인칭으로 작성됨. AI 양산 느낌 차단.

### 원칙 5: 자막 이모지 절대 금지
자영업자 전문성/신뢰감 손상. Claude 프롬프트에 hard rule + 후처리 검증으로 이중 차단.

이번 스펙에는 **YouTube 직접 업로드만** 포함. 다른 플랫폼은 각자 SEO 체계가 달라 단순 캡션 복제로 안 됨:

| 플랫폼 | SEO 체계 | 캡션 길이 | 해시태그 | 별도 스펙 |
|---|---|---|---|---|
| YouTube | 키워드 + 챕터 + 챕터타임코드 | ~5,000자 | #5~15 | ✅ 이번 |
| Instagram Reels | 메타 SEO + 위치태그 + 멘션 | ~2,200자 | #20~30 | 후속 |
| Threads | 짧은 문장 + 멘션 | ~500자 | 적음 | 후속 |
| Facebook Reels | 메타 SEO + 페이지 멘션 | ~1,000자 | 적음 | 후속 |
| TikTok | 트렌드 사운드 + 트렌드 해시태그 | ~150자 | #핵심3~5 | 후속 |
| 샤오홍슈 | 중국어 SEO + 위치 + 카테고리 | 한국어 SEO 불가 | 다른 체계 | 후속 |

**원소스 멀티유즈 전략 (별도 스펙):** 한 영상으로 6개 플랫폼 캡션을 동시 생성. 각 플랫폼별 SEO/해시태그/길이/톤 최적화. 자영업자가 한 번 만들면 6곳에 동시 게시 가능. 이 스펙은 이번 1차 출시 후 별도 디자인.

## 4. 사용자 플로우

### Step 1 — 입력 (한 화면)

```
┌─ 콘텐츠 ──────────────────────────────┐
│ ○ 블로그 글 사용                       │
│   [텍스트 영역 — /blog-writer 자동 입력] │
│ ○ 키워드만 사용                        │
│   [키워드 입력]                         │
└────────────────────────────────────────┘

┌─ 내 경험·느낌 (필수) ─────────────────┐
│ [자유 텍스트 — 본인의 구체적 경험]      │
└────────────────────────────────────────┘

┌─ 내 정체성 ──────────────────────────┐
│ 화자: [매장 사장 / 블로거 / 강사 /      │
│        컨설턴트 / 프리랜서 /            │
│        직접 입력]                       │
│ 톤: ( ● 전문가  ○ 친근한 친구 )         │
└────────────────────────────────────────┘

화자 페르소나 5종 (+직접 입력):
1. 매장 사장
2. 블로거
3. 강사
4. 컨설턴트
5. 프리랜서
6. 직접 입력 (자유)

(디자이너는 MVP에서 제외 — 프리랜서로 흡수 가능.
필요 시 후속 스프린트에서 확장)

┌─ 영상 길이 ─────────────────────────┐
│ ○ 30초  ● 45초  ○ 60초  ○ 90초      │
└──────────────────────────────────────┘

[ 다음: 벤치마킹 영상 찾기 → ]
```

**검증:**
- 블로그 글 또는 키워드 둘 중 하나는 필수
- 내 경험·느낌 필수 (50자 이상)
- 화자 페르소나 + 톤 선택 필수

### Step 2 — 벤치마킹 (시스템 자동, 약 30~45초)

진행 상태는 SSE로 실시간 표시 (섹션 16 참고).

1. **키워드 확장** (Gemini Flash, ~3초)
   - 블로그 글 또는 사용자 키워드를 입력으로 받아 3종류 키워드 생성:
     - **메인 키워드** (3~5개) — 글의 핵심 명사
     - **관련 키워드** (5~7개) — 의미적 인접, 동의어, 상위/하위 카테고리
     - **트렌드 키워드** (2~3개) — 현재 검색 인기와 결합
   - 메인 + 관련을 조합해 **검색 쿼리 5개** 생성
   - 예시 (블로그 글이 "신랑 정장 추천"인 경우):
     - 쿼리 1: "신랑 정장 추천"
     - 쿼리 2: "예비 신랑 패션"
     - 쿼리 3: "결혼식 슈트 코디"
     - 쿼리 4: "남자 정장 결혼"
     - 쿼리 5: "봄 결혼식 신랑 룩"

2. **YouTube 병렬 검색 + 필터** (~10초)
   - 5개 쿼리를 **병렬로 동시 검색**
   - 각 쿼리당 search.list로 50개씩 받음 → 총 250개 후보
   - dedupe (중복 영상 제거) → 약 150~200개 unique
   - 필터 통과:
     - 한국어 (regionCode=KR, relevanceLanguage=ko)
     - 영상 길이 ≤ 90초 (Shorts)
     - 자막 보유 (videoCaption=closedCaption)
     - 업로드 시점 ≤ 12개월
     - 조회수 ≥ 10,000
     - **조회수 / 구독자 비율 ≥ 5** (작지만 터진 영상 보장)
   - 필터 통과한 영상 50~100개 → viewToSubRatio 내림차순 정렬
   - **상위 5개 선정**

3. **결과 카드 표시** (Step 2.5 — 사용자 인터랙션)
   - 5개 카드를 그리드로 표시 (자세한 UX는 섹션 21 참고)
   - 1번 카드(최고 비율)는 **자동 ✓ 체크**
   - 사용자 옵션:
     - 그대로 진행 (자동 체크 1개로)
     - 1~3개 직접 선택
     - "벤치마킹 없이 바로 대본" 건너뛰기

4. **선택된 영상 깊은 분석** (Gemini Vertex AI Pro 사고 모드, ~20초)
   - 선택된 1~3개 영상 URL을 Gemini에 직접 전달
   - 멀티모달 분석 (영상 + 음성 + 자막)
   - 출력: 구조화된 JSON (Schema 5절 참고)
   - 영상별로 캐싱 (`bench:analyze:{videoId}` TTL 30일)

5. **패턴 카드 1장 표시**
   - aggregated 패턴 한국어 요약
   - 추천 프리셋 + advice
   - "이 패턴으로 대본 작성할게요" 확인 → Step 3

### Step 3 — 대본 생성 + 인라인 편집

- Claude Opus가 다음 입력으로 대본 생성:
  - 사용자 블로그 글 (베이스 콘텐츠)
  - 사용자 경험·느낌 (개인화 양념)
  - Gemini JSON (구조 가이드)
  - 화자 페르소나 + 톤
- 출력: 대본 JSON (scenes 배열)
- 사용자에게 표시: 텍스트 형태로 인라인 편집 가능
- 편집 후 "다음" 클릭 → Step 4

### Step 4 — 음성 + BGM

기존 숏폼 도구의 음성 단계 재사용. 단, BGM 선택 옵션 추가:
- 본인 음성 업로드 또는 TTS 선택 (Supertone/ElevenLabs)
- 미리듣기
- BGM: 카테고리 선택 (활기찬/차분/감성/일상/없음)

### Step 5 — 비주얼 액센트 (간단)

```
┌─────────────────────────────────────────┐
│ 메인 영상은 키네틱 타이포로 자동 생성돼요│
│ 핵심 순간에 사진/이미지를 양념으로 추가  │
│ 하면 더 풍성해져요. (선택사항)            │
└─────────────────────────────────────────┘

┌─ 내 사진 (보관함에서) ─┐
│ [+ 사진 선택] 0/3장    │
│ 추천: 매장 사진, 본인  │
│ 사진, 메뉴/상품 사진    │
└────────────────────────┘

┌─ AI 이미지 (자동 생성) ─┐
│ ○ 사용 안 함            │
│ ● 1장 생성              │
│ ○ 2장 생성              │
└─────────────────────────┘

[ 다음: 미리보기 → ]
```

- 0~3장 사진 + 0~2장 AI 이미지
- 사진은 카드뉴스에서 만든 `<ImagePickerModal>` 재사용 (보관함 통합)
- AI 이미지는 기존 FLUX Schnell 파이프라인 재사용

### Step 6 — 미리보기 + 커스터마이징 ⭐ (가장 비중 큰 단계)

**진입 시 자동 추천:**
```
┌────────────────────────────────────────┐
│ 벤치마킹 영상 패턴 분석:                 │
```

## 11. Gemini Vertex AI 모델 선택 + 비용

### 모델 선택: **Gemini Pro 사고(Thinking) 모드**

벤치마킹 영상 분석은 다음 이유로 **최상위 Pro 사고 모드** 사용:

1. **캐시 30일** — 같은 영상은 한 번만 분석되므로 1회 비용을 N명이 분담 → 1인당 비용 거의 0
2. **다운스트림 품질 결정** — Gemini 패턴 추출 정확도 → Claude 대본 → 최종 영상의 품질이 모두 이 단계에 의존
3. **자영업자 신뢰** — 분석 결과가 신뢰 가지 않으면 전체 도구 사용 안 함
4. **운영자 Vertex 크레딧 보유 중** — 사용 안 하면 만료

**구체적 모델 ID:** Vertex AI 콘솔에서 사용 가능한 **최상위 Gemini Pro thinking 모델** (구현 단계에서 확정).
- 우선 후보: `gemini-3.0-pro-thinking` (있으면)
- 폴백: `gemini-2.5-pro` (사고 모드 기본 탑재)
- 폴백 2: `gemini-2.0-pro`

플래시(Flash) 모델은 검색 키워드 추출(Step 2 첫 단계)에만 사용 — 단순 텍스트 작업이라 Flash로 충분.

### 1회 영상 분석 비용 (Pro 사고 모드 추정)

- 60초 영상 ≈ 15,500 input tokens (영상 토큰화율 258 tokens/sec)
- 사고 모드 토큰 ≈ 2,000 tokens
- 출력 ≈ 500 tokens
- 1건 비용: ~$0.015 (Flash 대비 10배)
- 3건 분석: **약 $0.045**

### 1년 비용 시뮬레이션

## 16. 실시간 진행 표시 (SSE) — P0 추가

벤치마킹/생성이 30~90초 걸리므로 진행 상태 실시간 표시 필수.

### 구현 방식: Server-Sent Events (SSE)

WebSocket보다 단순. 단방향 푸시면 충분.

```
GET /api/shortform-progress?jobId=xxx
Content-Type: text/event-stream
```

### 진행 단계별 이벤트

```
event: step
data: { "step": "keyword-extraction", "status": "running", "progress": 0 }

event: step
data: { "step": "keyword-extraction", "status": "done", "progress": 100 }

event: step
data: { "step": "youtube-search", "status": "running", "progress": 30 }

event: step
data: { "step": "youtube-search", "status": "done", "result": { "candidates": 5 } }

event: step
data: { "step": "video-analysis", "status": "running", "subStep": "video-1", "progress": 50 }

event: complete
data: { "result": { ... } }

event: error
data: { "error": "...", "step": "..." }
```

### 클라이언트 UI

```
┌────────────────────────────────────────┐
│ 벤치마킹 진행 중...                     │
│                                          │
│ ✅ 키워드 추출           완료           │
│ ✅ 후보 영상 5개 검색    완료 (5/5)     │
│ ⏳ 영상 1번 분석 중...   ⠋ 70%          │
│ ⏸  영상 2번 대기 중                     │
│ ⏸  영상 3번 대기 중                     │
│                                          │
│              [ 취소 ]                    │
└────────────────────────────────────────┘
```

### 백엔드 구조

- 백그라운드 작업 큐 (Vercel Function 안에서 async)
- Redis pub/sub로 진행 상태 전달
- SSE 엔드포인트가 Redis 구독 → 클라이언트로 푸시
- jobId는 UUID, Redis key: `job:progress:{jobId}` TTL 1시간

## 17. 취소/중단 처리 — P1 추가

진행 중인 작업을 안전하게 멈출 수 있어야 함.

### 사용자 액션

- "취소" 버튼 → POST `/api/shortform-cancel?jobId=xxx`
- 백엔드: Redis에 `job:cancel:{jobId} = 1` 설정
- 진행 중인 단계가 다음 체크포인트에서 cancel 플래그 확인 → throw CancelledError
- finally 블록에서 정리 (Vertex AI 호출 중이면 abort, 캐시 정리 등)

### 크레딧 환불 정책

| 취소 시점 | 환불 |
|---|---|
| Step 1~5 (대본 생성 전) | 100% (아직 차감 안 됨) |
| Step 6 (미리보기) | 100% (아직 영상 렌더 안 함) |
| Step 7 진입 (영상 생성 시작) | 진행률 기반 부분 환불 |
| Step 7 완료 후 (다운로드 직전) | 환불 없음 (이미 결과물 생성됨) |

크레딧 차감은 **Step 7 영상 렌더 시작 시점**에만 발생. 그 전까지는 무료로 진행 가능.

### 백그라운드 모드

사용자가 브라우저 탭 닫아도 계속 진행 → 완료 시 브라우저 알림 + 마이페이지 "내 영상"에 표시. SSE 끊겨도 백엔드 작업은 계속.

## 21. Step 2 벤치마킹 영상 노출 UX (디자인 디테일)

벤치마킹 영상을 사용자에게 노출하는 것은 **가장 큰 마케팅 가치**. 안 보여주면 우리 차별화가 시각화되지 않음.

### 노출의 5가지 가치

1. **신뢰** — 자영업자는 AI 블랙박스 거부감 큼. 출처 명확 → 신뢰 ↑
2. **마케팅 시각화** — "벤치마킹 도구"라고 100번 말하는 것보다 5개 카드 띄워주는 게 100배 강력
3. **저작권 투명성** — "참고 영상 5개" 명시 → 표절 시비 차단
4. **사용자 학습 효과** — 사장님이 영상 보면서 본인 영감 → 도구 = 단순 자동화 아닌 교육 가치
5. **결과 만족도** — 사용자가 직접 고른 영상 기반 → "내가 골랐으니까" 자기 수용

### UX 디자인 원칙

노출하되 **귀찮음 최소화**가 핵심. 30초 안에 다음 단계로 갈 수 있어야 함.

```
┌────────────────────────────────────────────┐
│ Step 2: 벤치마킹 영상                        │
│                                              │
│ ✨ "신랑 정장" 관련 잘 터진 숏폼 5개를       │
│    찾았어요. 골라주세요 (1~3개)              │
│                                              │
│ ┌────┬────┬────┬────┬────┐                │
│ │ ☑  │ ☐  │ ☐  │ ☐  │ ☐  │  자동 체크     │
│ │썸네 │썸네 │썸네 │썸네 │썸네 │                │
│ │ 8x │ 6x │ 5x │ 5x │ 5x │  view/sub      │
│ │ 12 │ 8.4│ 5.2│ 5.0│ 5.0│  만 조회       │
│ │신부 │결혼 │웨딩 │신랑 │드레 │  채널명        │
│ │수업 │준비 │플래 │룩북 │스샵 │                │
│ └────┴────┴────┴────┴────┘                │
│                                              │
│ ↑ 호버 시 첫 5초 자막 미리보기 표시          │
│                                              │
│ [ 선택한 영상으로 분석 → ] (1개 선택됨)      │
│                                              │
│ [ 벤치마킹 없이 바로 대본 만들기 → ]         │
│   (작은 부가 버튼)                            │
└────────────────────────────────────────────┘
```

### 카드 정보 우선순위

각 카드에 표시할 정보 (작은 카드 안에 압축):

1. **썸네일** (가장 큰 비중)
2. **viewToSubRatio** (큰 숫자, "8x" 형태) — 우리 핵심 지표
3. **조회수** (간단, "12만")
4. **채널명** (작게)
5. **호버 시 펼쳐지는 정보**:
   - 첫 5초 자막 미리보기
   - 정확한 구독자 수
   - 영상 길이
   - 업로드 날짜

### 자동 체크 정책

- 5개 카드 중 **viewToSubRatio가 가장 높은 1개에 자동 ✓**
- 사용자가 그냥 "다음" 누르면 1개로 진행
- 더 정밀한 결과 원하면 추가 체크 (최대 3개)

### "건너뛰기" 옵션

- 작은 보조 버튼 "벤치마킹 없이 바로 대본 만들기"
- 클릭 시: 폴백 모드로 Step 3 진입
- 폴백 모드: Claude가 사용자 입력만으로 대본 작성, 프리셋은 사용자 톤 선택에 따라 자동

### 결과 0~4건일 때

- 0건: "관련 영상을 찾지 못했어요. 키워드를 더 일반적으로 바꿔보세요" + 폴백 모드 버튼
- 1~4건: 그대로 표시 + 안내 ("기준에 맞는 영상이 N개입니다. 더 광범위한 검색을 원하면 키워드를 일반화하세요")

### 결과 카드 캐싱

- Step 2의 결과는 프로젝트 draft에 저장
- 사용자가 다시 진입해도 같은 5개 카드 표시 (동일성 보장)
- "다시 검색" 버튼으로 새로 불러오기 가능 (캐시 무시)

## 27. 성공 기준

### 핵심 기능 (P0)
- [ ] 자영업자가 키워드/블로그 글 + 본인 정체성 입력 → 60초 내에 대본+영상+캡션 패키지 받음
- [ ] 키워드 5개 병렬 검색으로 벤치마킹 영상 5개 이상이 표시됨 (95% 케이스)
- [ ] Gemini 패턴 추출 JSON이 schema 검증을 통과 (99% 케이스)
- [ ] 생성된 대본에 이모지 0개, 일반론 표현 ≤ 2회
- [ ] 캡션이 벤치마킹 패턴의 해시태그 수 ±2 이내로 일치
- [ ] Step 6 미리보기에서 모든 커스터마이징이 실시간 반영
- [ ] 폴백 모드 (벤치마킹 없이) 단독으로도 대본 생성 가능

### Retention 기능 (P0 추가)
- [ ] 브랜드 킷 한 번 저장 → 모든 영상에 자동 적용
- [ ] 프로젝트 히스토리에서 어제 만든 draft를 이어서 작업 가능
