# 결제 동선 디자인 리뉴얼 스펙

> 목표: 결제 승인(~1주 내) 전에 "돈 내고 쓸 만한 서비스" 느낌을 주는 수준으로 핵심 동선 품질 업스케일
> 기간: 2주 (2026-04-02 ~ 04-16)
> 제약: shortform.html은 Codex Remotion 전환 중이므로 건드리지 않음

---

## 1. 현재 문제 진단

### 공통: "AI가 만든 것 같은 느낌"
- 전 페이지 동일한 다크 네이비 히어로 + 동일 레이아웃 = 개성 없음
- 이모지 남발, hover-lift 버튼, 떠있는 카드 = ChatGPT 랜딩 클리셰
- Noto Sans KR 단독 사용, 타이포그래피 위계 부족

### index.html (랜딩)
- 히어로: `#1A1A2E → #16213E → #0F3460` 3단 그라데이션 + 초록/파랑 방사형 오버레이
- Chapter 1/2/3 색상 코딩 카드 = 전형적 AI 템플릿 구조
- CTA: 초록 그라데이션 전체 너비 버튼 = 뻔함
- 도구 8개의 가치 전달이 약함 — "뭘 해주는 서비스인지" 3초 안에 안 와닿음

### blog-writer.html (핵심 유료 도구)
- 기능 자체는 탄탄: 글 유형/톤/CTA 선택, AI 검수기(90점 만점), 자동 수정
- 히어로가 다른 페이지와 똑같아서 "프리미엄 도구" 차별감 없음
- AI 검수기가 핵심 유료 가치인데 스크롤 아래 묻혀있음
- 결제 가치 명시 없음 (1크레딧 = 얼마인지 현재 페이지에서 모름)

### pricing.html (결제)
- 단일 상품(30크레딧/9,900원)만 존재 — 가성비 비교 기준 없음
- "결제하기" 버튼 → "준비 중" 모달 (결제 승인 전이므로 정상이나, 승인 후 즉시 연결 필요)
- 소셜 프루프 전무 (후기, 사용자 수, 생성 건수 등)
- "무료 체험 3회"가 FAQ 하단에 묻혀있음
- 오픈톡방 20% 보너스 배너는 좋은 요소

## 2. 디자인 방향

### 원칙
1. **"사람이 정성 들여 만든" 느낌** — AI 템플릿 클리셰 탈피
2. **결제 동선 집중** — 랜딩 → (가입) → 도구 체험 → 결제 흐름만
3. **기능 추가 없음** — 시각/문구/레이아웃 개선만. 새 기능 YAGNI
4. **기존 기능 보존** — JS 로직, API 연동 절대 안 건드림

### 디자인 톤 전환
| 항목 | AS-IS | TO-BE |
|------|-------|-------|
| 히어로 | 다크 네이비 그라데이션 (전 페이지 동일) | 페이지별 차별화. 라이트 톤 기반 + 포인트 컬러 |
| 색상 | 오렌지 #ff5f1f 단일 액센트 | 오렌지 유지하되 보조 컬러 추가, 이모지 대폭 축소 |
| 카드 | box-shadow 떠있는 카드 | 미세한 보더 + 배경색 차이로 구분, 그림자 최소화 |
| 타이포 | Noto Sans KR 단일, 위계 약함 | 웨이트/사이즈 위계 강화 (900 헤드라인, 400 본문) |
| CTA | 전체 너비 그라데이션 버튼 | 적절한 너비, 단색 or 미세 그라데이션, 명확한 액션 텍스트 |
| 이모지 | 과다 사용 | 필수 아이콘만 남기고 제거 |

## 3. 페이지별 상세 스펙

### Phase 1 (1~2일): 핵심 3페이지

#### 3-1. index.html — 랜딩

**목표**: 3초 안에 "블로거를 위한 AI 도구 모음"이 와닿게

- 히어로: 라이트 배경 전환. "뚝딱툴"이 뭔지 한 줄 + 핵심 가치 3개 불릿
- 도구 쇼케이스: 8개 도구를 카테고리별 그리드로 재배치 (무료 도구 / PRO 도구 구분)
- 사회적 증거: 가입자 수 또는 생성 건수 카운터 (Redis에서 실시간 조회 가능)
- CTA: "무료로 시작하기" → signup.html (이미 로그인이면 guide.html)
- 이모지/뱃지 대폭 축소

#### 3-2. blog-writer.html — 블로그 글 생성기

**목표**: "이건 돈 내고 쓸 만하다" 프로 도구 느낌

- 히어로: index와 차별화된 디자인. AI 검수기 + 자동 수정 기능을 히어로에서 강조
- 입력 UI: 기존 스텝 구조 유지, 카드 디자인만 정돈
- 결과 영역: 현재 구조 유지 (이미 잘 되어있음)
- 가격 힌트: 입력 영역 근처에 "1크레딧으로 고품질 블로그 글 생성" 문구
- AI 검수기: 결과 최상단으로 배치 강조 (현재 하단 묻힘)

#### 3-3. pricing.html — 결제

**목표**: 신뢰감 + 전환율

- 가격 카드: 현재 단일 상품 유지하되, "1크레딧당 330원" 단가 명시
- 무료 체험 강조: 상단에 "회원가입만 하면 블로그 글 3회 + 이미지 3회 무료" 배너
- 크레딧 사용처: 도구별 차감표를 시각적으로 개선 (아이콘 + 간결한 표)
- 소셜 프루프: 가입자 수, 총 생성 건수 (Redis 기반)
- FAQ: 현재 3개 유지, 배치만 개선
- 결제 버튼: 토스페이먼츠 승인 후 즉시 연결 가능하도록 준비

### Phase 2 (3~4일): 전환 흐름 + 품질

#### 3-4. login.html / signup.html — 가입 흐름
- Phase 1 디자인 톤에 맞춰 통일
- 가입 폼: 불필요한 시각 요소 정리, 깔끔하게
- 약관 동의 UI 개선

#### 3-5. guide.html — 사용법
- 도구별 사용법을 탭 또는 아코디언으로 정리
- 스크린샷/GIF 추가 검토 (있으면 좋지만 없어도 진행)

#### 3-6. 블로그 글 생성기 퀄리티 튜닝
- api/generate.js 프롬프트 점검 (현재 temperature 0.5)
- AI 검수기 7개 기준 가중치 재검토
- replaceAIVocabulary() 매핑 17개 → 추가 필요한 단어 있는지 확인

#### 3-7. 공통 네비바/푸터
- 리뉴얼 디자인 톤에 맞춰 네비바 스타일 업데이트
- 푸터: 사업자 정보 유지, 디자인만 정돈

### Phase 3 (남는 시간): 디테일 패스

#### 3-8. 에러/로딩 상태 UX
- 생성 중 로딩 인디케이터 개선 (현재 스피너 → 진행률 또는 단계 표시)
- API 에러 시 사용자 친화적 메시지
- 네트워크 끊김 등 엣지 케이스

#### 3-9. 반응형 점검
- 모바일에서 핵심 3페이지 동작 확인
- 네비바 모바일 메뉴 (navbar-mobile.css) 점검

#### 3-10. 전체 일관성 마무리
- 색상/간격/폰트 크기 전 페이지 통일
- 불필요한 코드/주석 정리 (/simplify)

## 5. 건드리지 않는 것 (명시적 제외)

- shortform.html — Codex Remotion 전환 중
- card-news.html — 정상 동작, 결제 동선 아님
- blog-image.html / blog-image-pro.html — 이미지 깨짐은 별도 과제
- column-*.html (32편) — SEO 콘텐츠, 디자인 변경 불필요
- api/*.js 로직 — 프롬프트 튜닝(3-6) 제외하고 서버 코드 안 건드림
- 새 기능 추가 — 없음. 시각/문구/레이아웃 개선만

## 7. 성공 기준

1. index.html에 처음 방문한 사람이 3초 안에 "블로거용 AI 도구"임을 인지
2. blog-writer.html이 "무료 도구"가 아닌 "프로 유료 도구" 느낌
3. pricing.html에서 "9,900원이면 해볼 만하네" 전환 유도
4. 전 페이지에서 "AI가 만든 사이트" 느낌 탈피
5. 결제 승인 시 즉시 토스페이먼츠 연결 가능한 상태
