# 황금키워드 찾기 설계서

## 1. 개요

자영업자, 블로거, 쇼핑 어필리에이터가 자신의 분야에서 **검색량은 높고 경쟁은 낮은 "황금키워드"**를 찾을 수 있는 도구.

### 사용자 흐름
```
입력 → AI 시드키워드 생성 → 네이버 API 확장 → 황금키워드 점수 산출 → 결과 표시
```

## 2. 입력 UI

### 필수 입력 (4개 필드)

| 필드 | placeholder 예시 | 설명 |
|------|-----------------|------|
| 내 분야 | 웨딩컨설팅, 육아/리빙, 카페 창업 | 업종 또는 블로그 카테고리 |
| 나는 | 웨딩플래너, 육아 블로거, 카페 사장 | 사용자의 역할/입장 |
| 타겟 독자 | 예비 신부, 초보 엄마, 예비 창업자 | 콘텐츠를 읽을 사람 |
| 자주 받는 질문 | "스드메 비용이 얼마예요?" (선택, 여러 줄) | 실제 고객/독자 질문 → 시드키워드 품질 향상 |

### UI 구성
- 페이지: `keyword-finder.html` (독립 페이지)
- 네비게이션: "황금키워드" 메뉴 추가
- 디자인: 기존 생성기와 동일한 라이트 테마 + 오렌지 액센트
- 로그인 필수, 무료 횟수: 3회/일

## 3. 처리 파이프라인

### Phase 1: AI 시드키워드 생성

**Claude Haiku** 호출로 사용자 입력 기반 시드키워드 20~30개 생성.

```
입력: 분야 + 역할 + 타겟 + 질문
출력: ["웨딩 비용", "스드메 추천", "본식 드레스", ...] (20~30개)
```

프롬프트 전략:
- 타겟 독자가 검색할 법한 키워드
- 정보성 키워드 (방법, 추천, 비용, 후기 등)
- 롱테일 키워드 포함 (3~5 어절)
- 질문형 키워드 포함 ("~하는 법", "~차이")

### Phase 2: 네이버 API 키워드 확장

시드키워드 20~30개를 4개 API로 확장하여 후보 키워드 풀 생성.

#### 2-1. 검색광고 API — 연관 키워드 + 검색량
- **엔드포인트**: `GET /keywordstool` (RelKwdStat)
- **입력**: 시드키워드 (최대 5개씩 배치)
- **출력**: 연관키워드 목록 + 월간 PC/모바일 검색수 + 경쟁 정도(높음/중간/낮음)
- **활용**: 핵심 데이터 소스. 검색량 + 경쟁도가 황금키워드 점수의 기반

#### 2-2. 검색 API — 자동완성 키워드
- **엔드포인트**: 네이버 검색 자동완성
- **입력**: 시드키워드
- **출력**: 자동완성 제안 키워드 목록
- **활용**: 실제 사용자가 검색하는 롱테일 키워드 발굴

#### 2-3. 검색어 트렌드 API (DataLab)
- **엔드포인트**: `POST /v1/datalab/search`
- **입력**: 상위 후보 키워드 (최대 5개 그룹, 각 20개)
- **출력**: 기간별 검색 트렌드 (1~12개월)
- **활용**: 상승 트렌드 키워드 가산점, 하락 트렌드 감점

#### 2-4. 쇼핑인사이트 API
- **엔드포인트**: `POST /v1/datalab/shopping/categories`
- **입력**: 카테고리 키워드
- **출력**: 쇼핑 검색 트렌드, 카테고리별 인기 검색어
- **활용**: 쇼핑 연관 키워드 발굴 + 구매 의도 키워드 가산점

### Phase 3: 황금키워드 점수 산출

#### 점수 공식 (100점 만점)

```
황금점수 = 검색량 점수(30) + 경쟁도 점수(20) + 포화도 점수(20) + 트렌드 점수(20) + 보너스(10)
```

| 요소 | 배점 | 산출 방법 |
|------|------|----------|
| 검색량 | 30점 | 월간 검색수 구간별 (100~1000: 22pt, 1000~5000: 30pt, 5000~: 25pt) |
| 경쟁도 | 20점 | 낮음: 20pt, 중간: 10pt, 높음: 3pt |
| 포화도 | 20점 | 블로그 발행량/월간 검색수 비율 (낮을수록 고득점) |
| 트렌드 | 20점 | 상승: 20pt, 유지: 10pt, 하락: 0pt |
| 보너스 | 10점 | 쇼핑 연관(+5), 질문형(+3), 롱테일 3어절+(+2) |

#### 포화도 지수 (블로그 발행량 / 월간 검색수)
- 포화도 0~5: 블루오션 (20pt)
- 포화도 5~15: 적정 경쟁 (15pt)
- 포화도 15~30: 보통 (10pt)
- 포화도 30~50: 경쟁 과열 (5pt)
- 포화도 50+: 레드오션 (0pt)

#### 검색량 최적 구간 설명
- 100 미만: 너무 적음 (10pt)
- 100~1000: 틈새 시장 (30pt)
- 1000~5000: 최적 (40pt) ← 황금 구간
- 5000~10000: 경쟁 치열해지기 시작 (35pt)
- 10000 초과: 레드오션 가능성 (25pt)

## 4. 결과 UI

### 결과 카드 구성

#### 상단: 황금키워드 TOP 10
- 순위, 키워드, 황금점수(100점), 월간 검색수, 경쟁도(높/중/낮 뱃지), 트렌드(상승/하락 화살표)
- 각 카드 클릭 시 상세 정보 펼침

#### 하단: 전체 키워드 테이블
- 정렬 가능: 점수순, 검색량순, 경쟁도순
- 필터: 경쟁도(낮음만), 검색량 범위, 트렌드(상승만)
- CSV 다운로드 버튼

#### 키워드 상세 (펼침)
- 검색량 차트 (12개월 트렌드)
- 경쟁도 상세 (PC/모바일 비율)
- 추천 블로그 제목 2개 (AI 생성)
- "이 키워드로 블로그 글 쓰기" 버튼 → blog-writer.html로 연결

## 5. API 설계

### 엔드포인트: `POST /api/keywords.js`

#### Request
```json
{
  "field": "웨딩컨설팅",
  "role": "웨딩플래너",
  "target": "예비 신부",
  "questions": "스드메 비용이 얼마예요?\n본식 드레스 어떻게 고르나요?"
}
```

#### Response
```json
{
  "keywords": [
    {
      "keyword": "스드메 비용",
      "score": 87,
      "monthlySearch": 2400,
      "competition": "low",
      "trend": "rising",
      "trendData": [65, 70, 72, 80, 85, 90],
      "breakdown": { "search": 40, "competition": 30, "trend": 15, "bonus": 2 }
    }
  ],
  "totalFound": 150,
  "seedKeywords": ["웨딩 비용", "스드메 추천", ...],
  "remaining": 2,
  "limit": 3
}
```

### 처리 시간 예상
- Phase 1 (Haiku): ~2초
- Phase 2 (네이버 API 4종): ~5~8초 (배치 병렬 호출)
- Phase 3 (점수 산출): <1초
- **총 예상: 8~12초** → 프로그레스 바 필수

### Rate Limit
- 로그인 회원: 3회/일 무료
- 관리자: 무제한
