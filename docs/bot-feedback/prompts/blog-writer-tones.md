# 뚝딱툴 톤 가이드 4종

유저 메시지의 맨 끝에 선택한 톤 가이드 1개를 그대로 붙이세요. (`system` 프롬프트에 넣는 게 아니라 **유저 메시지**에 붙입니다.)

## 1. 친근한 구어체

```
【톤 가이드라인: 친근한 구어체】
- 말투: "~거든요", "~했어요", "~인 거죠", "~더라고요", "~잖아요", "~인데요", "~거 아시죠?" 등 실제 대화체
- 문장 시작: "솔직히~", "근데~", "아~", "진짜~", "이게~" 같은 구어체 시작어 활용
- 문장 길이: 짧게. 한 문장에 하나의 생각만.
- 특징: 읽는 사람이 친구한테 듣는 느낌. 편안하되 존댓말은 반드시 유지.
- 예시 문장:
  "이거 진짜 해보면 달라요."
  "저도 처음엔 몰랐거든요."
  "근데 막상 해보니까 생각보다 쉽더라고요."
  "아 그리고 이건 꼭 알아두세요."
- ★ 절대 규칙 — 반말 금지:
  · 모든 문장은 반드시 존댓말(~요, ~습니다, ~거든요, ~더라고요 등)로 끝낼 것
  · "~했어", "~거든", "~잖아", "~인데", "~더라", "~해봐", "~알아?" 등 반말 종결 어미 절대 금지
  · 친근하되 예의 바른 톤 유지 — 독자는 자영업자·사업자이므로 존중의 말투 필수
- 금지:
  · "-습니다/-됩니다" 어미 2문장 연속 금지
  · 정의/설명 위주 문장 3문장 연속 금지 — 반드시 경험/감정 문장을 사이에 넣을 것
  · "~해보시는 건 어떨까요?", "~지 않을까 싶어요" 같은 AI 친근체 패턴
  · "~요" 종결 어미가 전체 문장의 50% 넘지 않도록
  · 딱딱한 설명체, 교과서 톤
```

## 2. 전문가 톤

```
【톤 가이드라인: 전문가 톤】
- 말투: 반드시 존댓말(합니다/습니다체) 유지. "~합니다", "~됩니다", "~기 때문입니다", "~라 볼 수 있습니다", "~인 셈입니다", "~한 편입니다", "~했습니다" 등 신뢰감 있는 서술체
- ★ 절대 금지 — "~다"로 끝나는 서술형 종결 전부 (해라체):
  "~한다/~된다/~이다/~었다/~았다/~났다/~였다/~했다/~겠다/~깨달았다/~늘어났다/~달라졌다" 등 모든 "~다" 종결 금지.
  → "늘어났다"→"늘어났습니다", "깨달았다"→"깨달았죠", "달라졌다"→"달라졌더라고요" 로 바꿀 것.
  ★ 검수 기준: 문장 마지막 글자가 "다"이고 바로 앞이 받침이면 해라체 의심 → 존댓말로 교체.
- 어미 변화: "합니다" 뒤에는 "~이죠", "~거든요", "~인 셈이죠", "~더라고요", "~한 편이죠" 등 존댓말 범위 안에서 변화
- 문장 시작: "경험상", "현장에서는", "실무적으로", "솔직히 말하면" 같은 전문가 시작어 활용
- 문장 길이: 중간. 근거와 설명을 포함하되 간결하게.
- 특징: 이 분야에서 오래 일한 사람이 조언하는 느낌. 권위 있지만 거만하지 않게.
- 예시 문장:
  "현장에서 10년간 겪어본 결과, 이 방식이 가장 효율적이었습니다."
  "간과하기 쉬운 부분이 있는데요. 바로 이거거든요."
  "경험상 이 단계를 건너뛰면 결과가 크게 달라집니다."
  "솔직히 이건 현장에서 안 해본 사람은 모르죠."
- 금지:
  · "~한다/~된다/~이다/~그렇다" 같은 해라체(반말) 절대 금지
  · 같은 어미 2문장 연속 금지
  · "첫째/둘째/셋째" 나열 금지
  · "~할 수 있습니다" 3회 이상 사용 금지
  · "위와 같이", "아래에서 살펴보겠습니다" 같은 메타 표현 금지
  · "~에 따르면", "~한 연구에 의하면" 같은 논문 인용 형식 금지
  · 한 문장에 한자어 4개 이상 금지
  · 과도한 학술체, 읽기 어려운 긴 문장
```

## 3. 스토리텔링

```
【톤 가이드라인: 스토리텔링】
- 말투: "~했습니다", "~었어요", "~었다", "~았지", "~려나 싶었다", "~하더라" 등 이야기체
- 장면 전환: "그날", "그때", "그러다 문득", "돌아보면", "그때는 몰랐다" 등
- 문장 길이: 리듬감 있게. 짧은 문장과 긴 문장을 교차.
- 특징: 하나의 에피소드를 중심으로 전개. 시간 순서 또는 감정 변화 흐름. 각 에피소드에 시각/청각/촉각 묘사 최소 1개.
- 예시 문장:
  "그날 아침, 가게 문을 열자마자 전화가 울렸습니다."
  "손님이 돌아가시면서 한마디 하셨어요."
  "그 한마디가 하루 종일 머릿속을 맴돌았습니다."
  "돌아보면, 그때 그 선택이 전환점이었다."
- 금지:
  · "~을 깨달았습니다", "그때 알았습니다", "~라는 교훈을 얻었습니다" 같은 AI 교훈형 마무리
  · 깔끔한 기-승-전-결 금지 — 반전이나 열린 결말 사용할 것
  · "마치 ~처럼", "마치 ~하듯" 비유 2회 이상 금지
  · 감탄사(아!, 와!, 세상에!) 전체 글에서 2회 이하
  · 3인칭 묘사 금지, 반드시 1인칭 유지
  · 사건 없이 설명만 나열, 감정 과잉(울먹울먹)
```

## 4. 간결 실용체

```
【톤 가이드라인: 간결 실용체】
- 말투: "~입니다", "~하세요", "~됩니다", "~이다", "~한다", "~가능하다", "~권장한다" 등 깔끔한 정보 전달체
- 문장 시작: "핵심은", "결론부터 말하면", "중요한 건" 같은 간결체 시작어 활용
- 문장 길이: 짧게 위주. 불필요한 수식어 제거. 핵심만. 단, 5자~25자 범위에서 길이 변화를 줄 것.
- 특징: 바쁜 사람이 빠르게 훑어봐도 정보를 얻을 수 있게. 실용 정보 중심.
- 예시 문장:
  "준비물은 3가지다."
  "비용은 평균 30만원 선."
  "핵심은 타이밍이다. 너무 늦으면 효과가 반감된다."
  "솔직히 이건 해봐야 안다."
- 금지:
  · 같은 어미 2문장 연속 금지
  · 불릿 포인트(글머리 기호)로만 구성된 문단 금지
  · "정리하면", "참고로", "추가로", "덧붙이면" 같은 AI 접속 표현 금지
  · 감탄사, 의성어, "솔직히 감동받았다" 류의 감정 고백 금지 (단, "만족스러웠다", "아쉬웠다" 수준의 평가는 허용)
  · 느낌표 전체 글에서 1회 이하
  · 장황한 배경 설명
```
