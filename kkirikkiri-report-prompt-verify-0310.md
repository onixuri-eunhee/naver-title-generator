## 즉시 수정 — 높음 (5개)

### 1. 필드별 최소 글자수 합산 < 전체 최소 목표 (3개 프롬프트 모두)

| 프롬프트 | 필드별 최소 합 | 전체 최소 목표 | 갭 |
|---------|-------------|-------------|-----|
| 홈피드(A) | 1950자 (400+1400+150) | 2000자 | **-50자** |
| 네이버SEO(B) | 1900자 (350+1400+150) | 2000자 | **-100자** |
| 구글SEO(C) | 2050자 (250+1500+200+100) | 2200자 | **-150자** |

**문제**: AI는 "최소값"을 목표로 삼는 경향 → 각 필드를 최소치로 쓰면 전체 미달.
**수정 제안**:
- A: cta 최소 150→200자 (합산 2000자)
- B: hook 최소 350→400자 (합산 2000자)
- C: body 최소 1500→1650자 (합산 2200자)

### 2. "안녕하세요" 금지 vs 권위+정보형 패턴 충돌 (네이버SEO B)

- 금지 규칙 1번: "안녕하세요" 인사 금지
- 첫문장 패턴 ②: "안녕하세요. OO병원에서 10년간 근무하고 있는 OO의사입니다."

**수정 제안**: 금지 규칙을 "단독 인사만 금지 (전문가 자격+소개 동반 시 허용)"로 명확화.

### 3. 이미지 마커 형식 불일치 (C vs A/B)

- A/B: `(사진: OO 이미지 추천)`
- C: `(이미지: OO 사진, alt: 설명문)`

**문제**: 프론트엔드 마커 파싱 정규식이 두 형식을 모두 처리하는지 확인 필요. alt 생략 시 마커 미감지 가능.
**수정 제안**: 의도적 차이라면 코드 주석 명시 + 파싱 로직 확인.

### 4. 글자수 계산 기준 미명시

**문제**: 공백 포함/제외, 마커 텍스트 포함 여부 불명. 마커 8개 × ~15자 = ~120자가 포함되면 실제 "읽히는 콘텐츠"는 그만큼 짧아짐.
**수정 제안**: "마커 포함 전체 텍스트 기준" 또는 "순수 텍스트 기준" 명시.

### 5. B/C JSON hook 필드에 첫문장 패턴명 미기재

- A: "첫문장 패턴 4가지 중 택1: 에피소드/YES세트/성공사례/위험경고" ✅
- B: 패턴명 없음 ❌
- C: 패턴명 없음 ❌

**문제**: AI는 JSON 필드 설명을 우선시 → B/C에서 첫문장 패턴 규칙 무시 가능.
**수정 제안**:
- B: "경험답변/권위정보/YES세트/위험손해/질문 중 택1"
- C: "핵심답변수치/경험검증/위험손해/권위보강 중 택1"

## 개선 권장 — 중간 (14개)

### 6. "같은 접속사 글 전체 2번 이상 금지" — 비현실적
- 2000자+ 글에서 모든 접속사를 1회씩만 쓰는 것은 불가능
- **수정**: "같은 접속사 3문장 이내 반복 금지" 또는 "전체 3번 이상 금지"로 완화

### 7. "대박" 금지 vs 포지티브 원칙 "대박이었음" 예시 충돌
- **수정**: 포지티브 예시를 "이건 진짜 놀라웠음"으로 변경

### 8. "사람이 쓴 글의 특징" 항목 불균일
- A: 8개, B: 6개, C: 5개
- C에 "감각 묘사" 누락 (채널 무관 필수 요소)
- **수정**: 공통 필수 항목(실패담, 감각 묘사, 구체적 숫자, 감정 표현) 통일

### 9. 참여 유도 규칙이 A에만 있고 B에 누락
- 네이버 D.I.A 로직에서 참여도 중요 → B에도 추가 필요

### 10. 광고 금지 표현 목록 불일치
- A: "최고, 강추, 완전 추천, 대박"
- B: "최고, 강추, 가성비 甲"
- C: "최고, 강추, 최저가"
- **수정**: 합집합으로 통일

### 11. "같은 문법 구조 3문장 연속 금지" — A/B에 누락
- C에만 있는 유효한 AI 탐지 방지 규칙 → A/B에도 추가

### 12. hook 문장 수 vs 글자수 정합성
- C: 3~5문장에 300자 = 문장당 60~100자로 과도하게 긴 문장
- **수정**: C hook 문장 수를 "4~6문장"으로 조정 검토

### 13. 키워드 밀도 기준 "2000자" → 실제 최소 2200자
- C의 키워드 밀도가 2000자 기준인데 실제 최소는 2200자
- **수정**: "2200자 기준" 또는 횟수 기반 "메인 키워드 8~12회"로 변경

### 14. C 해시태그 "8~10개" → JSON 예시 10개 불일치
- **수정**: "10개"로 통일

### 15. 마커 필드 소속 모호 (대표이미지가 hook인지 body인지)
- **수정**: "body 필드 첫 줄에 대표이미지 마커" 명시

### 16. 톤 가이드 vs 포지티브 원칙 충돌 가능
- "간결 실용체"의 "감정 고백 금지" vs 포지티브 "감정 솔직 표현"
- **수정**: 포지티브 원칙에 "선택한 톤에 맞는 표현으로 적용" 단서 추가

### 17. "~적인" 분류에 "다양한/특별한" 잘못 포함
- "다양한", "특별한"은 "~적인" 형태 아님
- **수정**: "~적인 한자어(효과적인, 체계적인 등) + AI 전형적 형용사(다양한, 특별한, 풍부한)" 분리

### 18. FAQ 필드 소속 모호 (구글SEO C)
- body 안인지 별도 faq 필드인지 불명확
- **수정**: "FAQ는 별도 faq 필드에 작성"으로 명시

### 19. 규칙 순서 — 구조가 문체보다 뒤에 배치
- 현재: 글자수 → 문체(AI탐지) → 내용(첫문장) → 구조(소제목) → SEO
- 이상적: 글자수 → 구조 → 내용 → 문체 → 체크리스트
- 치명적이진 않으나 개선 시 AI 준수율 향상 가능
