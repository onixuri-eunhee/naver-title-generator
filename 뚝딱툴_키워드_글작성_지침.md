# 뚝딱툴 키워드 기반 글 작성 지침

> 키워드 전문가가 8편의 실전 피드백을 거쳐 검증한 패턴.
> 이 지침을 따르면 검색 유입 → 체류시간 → 내 분야 전환까지 한 흐름으로 연결된다.

---

## 1. 글 작성 전 필수: 검색의도 파악

### 왜 중요한가
같은 키워드도 검색하는 사람의 의도가 다르다.
검색의도를 모르고 글을 쓰면, 검색한 사람이 원하는 걸 안 주는 글이 된다.
→ 이탈률 높음 → 체류시간 짧음 → 검색 순위 하락 → 유입 0

### 검색의도 파악 방법 (STEP 0)
**글 작성 전 반드시 해당 키워드로 실제 검색하여 확인한다.**

1. 네이버/구글에 키워드 검색
2. 상위 노출 글 3~5개의 내용 확인
3. "이 키워드를 검색한 사람이 진짜 원하는 게 뭔지" 파악
4. 키워드의 실제 의미를 모르면 글이 성립하지 않음

> 실패 사례: "미리캔버스 디자인허브"를 템플릿 사용법으로 이해하고 글을 썼으나,
> 실제로는 기여자 수익 플랫폼이었음 → 초안 전체 폐기

### 검색의도 4가지 유형

| 유형 | 검색하는 이유 | 예시 |
|------|-------------|------|
| 정보형 | 기준·조건·구조를 알고 싶다 | 신용점수등급표, 아동수당2017년생 |
| 사용법형 | 어떻게 쓰는지 알고 싶다 | 쿠팡이츠사장님사이트 |
| 도구형 | 도구를 직접 쓰고 싶다 | 사대보험계산기, 퇴직금계산기 |
| 수익형 | 돈을 벌 수 있는지 알고 싶다 | 미리캔버스디자인허브, 틱톡스크롤돈버는방법 |

---

## 2. 홈피드용 vs 검색노출용 분류

검색의도를 파악한 후, **글에서 그 의도를 해결해줄 수 있는지**를 기준으로 분류한다.

| 판별 질문 | YES → 검색노출용 | NO → 홈피드용 |
|-----------|-----------------|--------------|
| 표면의도를 글에서 해결해줄 수 있는가? | 의도 충족 먼저 → 신뢰 → 전환 | 경험/공감 기반 → 퍼널 |

### 분류 예시

| 키워드 | 표면의도 | 글에서 해결? | 분류 |
|--------|---------|-------------|------|
| 만세력 계산기 | 사주 보고 싶다 | ⭕ 정확한 조건 설명 가능 | 검색노출용 |
| 쿠팡이츠사장님사이트 | 사용법 알고 싶다 | ⭕ 사용법 안내 가능 | 검색노출용 |
| 미리캔버스 디자인허브 | 수익 알고 싶다 | ⭕ 수익 구조 안내 가능 | 검색노출용 |
| 사대보험계산기 | 계산하고 싶다 | ❌ 계산기 제공 불가 | 홈피드용 |
| 틱톡스크롤돈버는방법 | 돈 벌고 싶다 | ❌ 벌게 해줄 수 없음 | 홈피드용 |

## 3. 제목 작성 규칙

### 절대 규칙
- **25자 내외**
- **타겟 제한 단어 절대 금지** — "자영업자", "사업자", "사장님", "프리랜서" 등을 제목에 넣지 않음
- 제목은 **모수 최대로 열기** → 글 안에서 퍼널로 좁히기
- 검색 키워드를 제목 앞쪽에 자연스럽게 포함

### 분류별 제목 패턴

**검색노출용:**
- 검색의도를 반영한 구체적 제목
- 숫자가 있으면 클릭률 상승

| 패턴 | 예시 |
|------|------|
| 키워드 + 정리/기준 | "신용점수 등급표 2026 기준, 내 점수가 어디쯤인지 한눈에 정리" |
| 키워드 + 틀릴 수 있는 N가지 | "만세력 계산기 결과가 틀릴 수 있는 3가지 경우" |
| 키워드 + 처음이면 여기부터 | "쿠팡이츠 사장님 사이트 접속부터 매출 확인까지, 처음이면 여기부터" |
| 키워드 + 실제로 얼마 | "미리캔버스 디자인허브 기여자 수익, 실제로 얼마나 되는지 정리" |

**홈피드용:**
- 경험/공감 기반 제목
- "해봤더니", "알게 된", "싶은 분들" 같은 경험 톤

| 패턴 | 예시 |
|------|------|
| 키워드 + 해보고 나서 알게 된 | "틱톡 스크롤로 돈 번다는 말, 해보고 나서 알게 된 현실" |
| 키워드 + 이게 맞나 싶은 | "사대보험계산기 돌려봤는데, 이 금액이 맞나 싶은 분들" |

### 제목 금지 사항
- ❌ "방법 / 꿀팁 / 비법" 단어
- ❌ "~하세요!" 강요형
- ❌ 타겟 제한 단어 (자영업자, 사업자, 사장님 등)
- ❌ 키워드 나열식 ("블로그마케팅/SNS마케팅/인스타마케팅")
- ❌ 성과 과장 ("매출 10배 상승!")

## 4. 글 구조: 5단 퍼널

### 공통 퍼널 원칙
```
제목: 모든 사람이 클릭 (넓은 입구)
  ↓
① 도입: 모든 사람이 공감
  ↓
② 잘못된 선택/놓치는 부분: 타겟만 공감하는 요소 삽입 → 비타겟 자연 이탈
  ↓
③ 구조·기준 제시: "이건 내 얘기다" → 타겟 확정
  ↓
브릿지: 이 글의 주제 → 내 분야로 동일 원리 연결
  ↓
④ CTA: 남은 독자 = 핵심 타겟
```

### 검색노출용 5단 구조

**① 도입 — 검색의도 충족**
- 표면의도를 바로 해결 (사용법, 조건, 절차, 수익 구조 등)
- 표, 목록 등으로 정보를 구조화
- 이 구간에서 전문성·신뢰 확보
- 검색의도가 충족되어야 체류시간이 길어짐

**② 대부분이 놓치는 부분 (브릿지 시작)**
- "여기서 끝나면 안 되는 이유" 제시
- 현장 경험 온도로 (매뉴얼체 금지)
- 타겟(자영업자·1인사업자·프리랜서)만 공감하는 요소 삽입

**③ 구조·기준 제시**
- 비교표로 정리 → 모바일 스크롤 유도 + 정보 정리 동시
- 독자가 "생각이 정리되는 느낌" 받게
- ③은 현실 정보로 끝내기 (방향 제시는 브릿지에서)

**③→④ 브릿지 (필수, 최소 한 문단)**
- "이 글의 주제를 이해하는 것 = 내 분야를 이해하는 것" 동일 원리 연결
- 운영자 경험 최소 한 문장 포함 ("저도 ~하면서")
- 상품 언급 시 "왜 만들었는가" 동기 문장이 먼저

**④ CTA**
- 판단을 독자에게 넘기기
- 가능성만 열기
- 댓글 유도 자연스럽게

### 홈피드용 5단 구조
- ① 도입: 경험/공감 기반 진입 (정보 제공 대신)
- ②③④: 검색노출용과 동일 원칙

## 5. 톤 & 문체 절대 규칙

### 금지 패턴과 대체 패턴

| 금지 | 이유 | 대체 |
|------|------|------|
| "~하는 게 훨씬 남는 장사입니다" | 훈계형 | "~로 쓰면 몇 백 원, ~로 쓰면 6개월. 어디에 쓸지는 각자 판단하는 거예요" |
| "이걸 모르고 하면 잃는 겁니다" | 주어가 사람 → 비난 | "순서가 바뀌면 먼저 나가는 방식이 됩니다" (주어를 상황으로) |
| "목적 없이 시작하면 끝난다" | 훈계형 | "목적이 정해져 있으면 흔들리는 일이 줄어듭니다" (긍정 전환) |
| "수수료 vs 검색유입, 뭐가 나을까" | 비교 강요 | "둘 다 하는 사장님들이 있습니다. 어디서부터 시작할지는 각자 상황이 다르니까요" |
| "그래서 제 강의를 들으시면..." | 억지 상품 등장 | "결국 직접 만들게 됐어요. 그게 ~를 만든 이유입니다" (동기 먼저) |
| "첫 번째, ~하는 경우. 두 번째..." | 매뉴얼체 | 겪는 순간을 그리기: "처음 고지서 받은 날, 금액 보고 한 번 더 확인하게 됩니다" |

### 핵심 원칙 3가지
1. **판단을 독자에게 넘기면 저항이 사라진다**
2. **주어를 "사람"에서 "상황"으로 바꾸면 훈계가 사라진다**
3. **설득이 아니라 구분선을 제시하면 독자가 스스로 판단한다**

## 6. 브릿지 작성 규칙

### 브릿지란?
③ 구조·기준 제시와 ④ CTA 사이에 들어가는 연결 문단.
"이 글의 주제"에서 "내 분야"로 자연스럽게 넘어가는 다리.

### 브릿지 필수 조건
- 최소 2~3문장, 한 문단 (한 문장이면 너무 얇다)
- 운영자 경험 최소 한 줄 포함
- 동일 원리 연결 공식 사용

### 브릿지 공식

```
[이 글의 주제]를 이해하는 것도 결국 [공통 원리]의 문제예요.
[내 분야]도 똑같습니다.
[공통 원리]를 먼저 아는 사람이 [결과 차이].
```

### 잘 된 브릿지 예시

**쿠팡이츠 → 마케팅:**
> "근데 그건 플랫폼이 시키는 일이에요.
> 내 가게를 직접 알리는 일은 따로 있습니다."

**사대보험 → 마케팅:**
> "보험료 구조를 잡는 것도 결국 나가는 돈의 흐름을 이해하는 것.
> 마케팅도 똑같습니다.
> 구조를 먼저 이해한 사람이 돈을 덜 씁니다."

**미리캔버스 → 마케팅:**
> "디자인 도구를 잘 쓰는 것과, 그 도구 위에 뭘 얹느냐는 별개의 역량이에요."

**신용점수 → 웨딩:**
> "3,500커플의 결혼을 함께하면서,
> 이 순서를 처음부터 아는 분은 거의 없었어요."

### 브릿지 금지

- ❌ ③에서 이미 방향을 제시하고, 브릿지에서 같은 말 반복 (같은 방향 3번 = "알겠어, ~하래잖아")
- ❌ 상품명이 글에서 처음 나올 때 동기 문장 없이 등장
- ❌ 운영자 경험 없이 CTA로 바로 점프

## 7. 상품 등장 규칙

상품(워크북, 강의, 툴 등)을 글에서 언급할 때의 규칙.

### 등장 순서 (필수)
```
① 전문 정보 제공으로 신뢰 확보
② "저도 ~하면서 이걸 수십 번 봤습니다" (경험 등장)
③ "그러다 보니 결국 직접 만들게 됐어요" (동기)
④ "그게 [상품명]을 만든 이유입니다" (상품 등장)
```

### 금지
- ❌ 신뢰 구축 없이 상품명 갑자기 등장
- ❌ "그래서 제 [상품]을 보시면..." (억지 연결)
- ❌ 글 전체에서 한 번도 언급 안 되다가 브릿지에서 갑자기 등장

---

## 8. 타겟 전환 (퍼널 좁히기) 규칙

### 원칙
넓은 독자 → 핵심 타겟으로 좁힐 때 **갑자기 점프하면 안 된다.**
중간에 자연 필터링 문장을 넣어서 비타겟이 스스로 이탈하게 만든다.

### 자연 필터링 패턴

**"여러 상황 중 가장 ~한 경우" 패턴:**
> "대출이 필요한 시점은 사람마다 달라요.
> 사업 자금일 수도 있고, 차 살 때일 수도 있고.
> 그중에서 한꺼번에 가장 많은 돈이 움직이는 시점이 있습니다.
> 결혼 준비예요."

**"여기까지 읽으셨다면" 패턴:**
> "여기까지 읽으셨다면, 아마 직장인은 아니실 거예요."

### 금지
- ❌ "자영업자라면..." (갑자기 타겟 호명)
- ❌ 도입에서부터 타겟을 한정 (모수 깎아먹기)

## 9. 체크리스트 (글 완성 후 검수)

### 검색의도
- [ ] 키워드로 실제 웹서치했는가?
- [ ] 표면의도를 글에서 해결해주고 있는가? (검색노출용)
- [ ] 홈피드/검색 분류가 맞는가?

### 제목
- [ ] 25자 내외인가?
- [ ] 타겟 제한 단어가 없는가?
- [ ] 키워드가 앞쪽에 포함돼 있는가?

### 퍼널
- [ ] ①에서 모든 사람이 공감하는가?
- [ ] ②에서 타겟만 공감하는 요소가 있는가?
- [ ] ③에서 비교표가 있는가?
- [ ] ③은 현실 정보로 끝나는가? (방향 제시 ❌)

### 브릿지
- [ ] 최소 한 문단(2~3문장) 이상인가?
- [ ] 운영자 경험이 포함돼 있는가?
- [ ] 동일 원리 연결 공식을 따르는가?
- [ ] 상품 등장 시 동기 문장이 먼저인가?

### 톤
- [ ] 훈계형 문장이 없는가?
- [ ] 비교 강요가 없는가?
- [ ] 매뉴얼체("첫 번째, ~하는 경우")가 없는가?
- [ ] 성과 과장 숫자가 없는가?
- [ ] 판단을 독자에게 넘기고 있는가?

### CTA
- [ ] 가능성만 열고 있는가?
- [ ] 같은 방향을 3번 반복하지 않는가?
- [ ] 댓글 유도가 자연스러운가?
